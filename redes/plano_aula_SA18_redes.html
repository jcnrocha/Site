Os Três Pilares do Endereçamento IP

🔢
IPv4
"4 octetos decimais separados por pontos · 32 bits · ~4,3 bilhões de endereços"
  • Estrutura: ex. 192.168.1.10
  • Cada octeto: 0 a 255 (8 bits = 2⁸ = 256 valores)
  • Total: 2³² = 4.294.967.296 endereços
  • Status: esgotado — solução temporária: NAT
🔒
Privado × Público
e NAT
"IPs privados são usados internamente e não roteiam na internet — o NAT traduz para um único IP público"
  • Privado A: 10.0.0.0/8
  • Privado B: 172.16.0.0/12
  • Privado C: 192.168.0.0/16
  • NAT: prolongou a vida útil do IPv4
🌐
IPv6
"128 bits hexadecimais · 2¹²⁸ endereços — praticamente inesgotável · futuro da internet"
  • Estrutura: 8 grupos de 4 hex separados por :
  • Ex.: 2001:db8::8a2e:370:7334
  • Coexistência com IPv4: dual stack e tunneling
  • Adoção lenta no Brasil — ainda em transição

Classes de Endereços IPv4

Classe Faixa do 1º octeto Uso típico
Classe A 1 – 126 Grandes redes · poucas redes, muitos hosts · ex.: corporações e provedores
Classe B 128 – 191 Redes de porte médio · universidades, empresas de médio porte
Classe C 192 – 223 Redes pequenas · muitas redes, poucos hosts · redes domésticas e empresariais comuns
Classe D 224 – 239 Multicast — transmissão simultânea para múltiplos destinos
Classe E 240 – 255 Reservado / experimental — não usado em produção

Conceitos Operacionais — O Que Aparece nas Configurações de Rede

🔧
Máscara de Sub-rede
Define qual parte do IP é rede e qual é host · ex.: /24 = 255.255.255.0 · 254 hosts
🚪
Gateway Padrão
"Portão de saída" da rede local · geralmente o IP do roteador · ex.: 192.168.0.1
🔄
DHCP
Atribui IP automaticamente ao conectar · ideal para usuários e dispositivos móveis
📌
IP Fixo
Configurado manualmente · sempre o mesmo endereço · servidores, impressoras, câmeras
🔀
NAT
Vários IPs privados → 1 IP público · roteador faz a tradução · prolongou a vida do IPv4

DHCP × IP Fixo — Quando Usar Cada Um

🔄
DHCP — Atribuição Automática
  • O servidor DHCP (geralmente o roteador) atribui o IP automaticamente
  • O dispositivo recebe IP, máscara, gateway e DNS sem configuração manual
  • O endereço pode mudar a cada conexão — é dinâmico
  • Ideal para computadores de usuários, notebooks, celulares e visitantes
  • Simplifica a administração em redes com muitos dispositivos
Use quando: o dispositivo não precisa ser localizado sempre no mesmo endereço
📌
IP Fixo (Estático)
  • O administrador configura manualmente o IP, máscara, gateway e DNS
  • O endereço nunca muda — sempre o mesmo, independente de reconexão
  • Facilita o acesso consistente ao dispositivo na rede
  • Ideal para servidores, impressoras de rede, câmeras IP e roteadores
  • Exige planejamento para evitar conflitos de endereço
Use quando: o dispositivo precisa ser encontrado sempre no mesmo endereço (serviços, acesso remoto)

💡 Reforço pedagógico: ao ver um endereço IP, o técnico deve conseguir identificar de imediato — privado ou público, qual classe, o que representa naquele contexto. Se possível, abrir o terminal e digitar ipconfig (Windows) ou ifconfig (Linux/Mac) para mostrar o IP, máscara e gateway do próprio computador da sala — conectando o conteúdo com a realidade imediata dos alunos.
