Questão 2 — Tecnologias de Conexão à Internet

  • A) FTTH
  • B) ADSL
  • C) Rádio WISP
  • D) DOCSIS
Questão 3 — Endereçamento IP
Um administrador de redes precisa configurar manualmente o endereço IP de um servidor de arquivos para que ele sempre seja encontrado no mesmo endereço dentro da rede local. Qual abordagem é mais adequada?
  • A) Configurar DHCP no servidor para receber IP automático
  • B) Atribuir um IP fixo (estático) manualmente ao servidor
  • C) Usar um endereço IPv6 público para o servidor
  • D) Configurar NAT no servidor para mascarar o endereço
Questão 4 — Protocolos de Aplicação

  • B) O servidor encontrou um erro interno ao processar a requisição
  • C) O recurso solicitado não foi encontrado no servidor
  • D) A página foi movida permanentemente para outro endereço

Um desenvolvedor percebe que sua aplicação consegue se conectar ao servidor (ping funciona e a porta está aberta), mas os dados recebidos chegam corrompidos com caracteres estranhos. Em qual camada do modelo OSI o problema provavelmente está ocorrendo?

🔑 Gabarito Oficial — Uso Exclusivo do Professor Gabarito · Critérios de Correção

Gabarito da Parte 1 — Questões Objetivas

QuestãoGabaritoJustificativa conceitual
Questão 1 D Cat6A STP possui blindagem individual por par — ideal para ambientes industriais com alta interferência eletromagnética · a fibra multimodo também seria imune, mas o Cat6A STP é a escolha mais prática e econômica para esse cenário.
Questão 2 C Rádio WISP (Wireless ISP) é exatamente a tecnologia utilizada por provedores regionais em cidades do interior — antenas em torres transmitem internet sem fio até os clientes · não exige infraestrutura de fibra ou cabo.
Questão 3 B Servidores devem ter IP fixo (estático) para garantir que sempre sejam encontrados no mesmo endereço — DHCP atribuiria um IP dinâmico que pode mudar, dificultando o acesso pelos clientes da rede.
Questão 4 C HTTP 404 = Not Found — o servidor foi acessado com sucesso, mas o recurso solicitado (página, arquivo, endpoint) não existe naquele endereço · 200 = OK · 500 = erro interno · 301 = redirecionamento.
Questão 5 C Dados corrompidos com caracteres estranhos indicam problema de codificação, tradução de formato ou criptografia — funções da Camada 6 (Apresentação) · as camadas inferiores estão ok pois a conexão foi estabelecida.

Critérios de Correção da Parte 2 — Questões Discursivas

Questão 6 — Fibra Óptica: Monomodo e Multimodo
Resposta esperada: Monomodo (SMF): núcleo estreito (~9 µm) · transmite um único raio de luz · longas distâncias (dezenas de km) · altíssima velocidade · uso: backbones, operadoras, interligação de cidades. Multimodo (MMF): núcleo mais largo (~50–62,5 µm) · transmite múltiplos raios · curtas distâncias (até ~500 m) · velocidade alta · uso: data centers, interligação de andares de prédios.

Critérios: diferença de núcleo (0,3 pt) · distância e velocidade de cada tipo (0,3 pt) · situações de uso corretas (0,4 pt).
Questão 7 — IPv4 e IPv6
Resposta esperada: Problema: IPv4 tem apenas ~4,3 bilhões de endereços (2³²) — esgotados. IPv4: 32 bits · 4 octetos decimais separados por pontos. IPv6: 128 bits · 8 grupos hexadecimais separados por dois pontos · 2¹²⁸ endereços. Por que a transição é lenta: custo de atualização, compatibilidade, soluções como NAT prolongaram a vida do IPv4.

Critérios: problema do esgotamento (0,2 pt) · estrutura IPv4 correta (0,2 pt) · estrutura IPv6 correta (0,2 pt) · diferença de capacidade (0,2 pt) · justificativa da transição lenta (0,2 pt).
Questão 8 — Three-Way Handshake
Resposta esperada: SYN: cliente envia um segmento SYN sinalizando que deseja iniciar uma conexão. SYN-ACK: servidor responde com SYN-ACK, confirmando o SYN do cliente e enviando seu próprio SYN. ACK: cliente confirma o SYN do servidor. Por que é necessário: garante que ambos os lados estão prontos para comunicar e sincroniza os números de sequência.

Critérios: cada etapa corretamente descrita (0,25 pt cada = 0,75 pt) · justificativa da necessidade do processo (0,25 pt).
Questão 9 — Tríade CIA
Resposta esperada: Confidencialidade: somente pessoas autorizadas acessam (ex: criptografia / ameaça: interceptação). Integridade: dados não foram alterados (ex: hash, assinatura / ameaça: alteração maliciosa). Disponibilidade: sistema acessível (ex: redundância, firewall / ameaça: DDoS).

Critérios: cada pilar com definição + ameaça + proteção corretas (0,33 pt cada ≈ 1,0 pt total).
Questão 10 — As 7 Camadas do Modelo OSI
Resposta esperada: 1 - Física (transmissão de bits/cabos); 2 - Enlace (MAC/switch); 3 - Rede (IP/roteador); 4 - Transporte (segmentação/TCP,UDP); 5 - Sessão (gerencia sessões/NetBIOS); 6 - Apresentação (tradução, criptografia/TLS); 7 - Aplicação (interface com usuário/HTTP, DNS).

Critérios: cada camada com nome + função + exemplo corretos · 0,14 pt por camada completa ≈ 1,0 pt total.
