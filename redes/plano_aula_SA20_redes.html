Os Pilares da Camada de Aplicação (TCP/IP)

🌐
HTTP / HTTPS
"Protocolo base da Web · HTTPS adiciona criptografia TLS/SSL para segurança e privacidade"
  • Request/Response (Cliente/Servidor)
  • Métodos: GET, POST, PUT, DELETE
  • Status: 200 (OK), 404 (Not Found)
  • Portas: HTTP (80) · HTTPS (443)
🔍
DNS & DHCP
"DNS traduz domínios em IPs · DHCP configura hosts dinamicamente ao conectar à rede"
  • DNS: Traduz nomes (ex: porta 53 UDP)
  • DNS Público: 8.8.8.8 (Google)
  • DHCP: Configura IP, máscara, gateway
  • DHCP Processo: DORA (portas 67/68 UDP)
📁
FTP / SFTP
"Protocolos de transferência de arquivos · SFTP implementa segurança utilizando criptografia SSH"
  • FTP: Sem criptografia (texto puro)
  • FTP portas: 21 (controle) e 20 (dados)
  • SFTP: Transferência segura (criptografia)
  • SFTP porta: 22 (túnel SSH)

Métodos HTTP Principais (Verbos HTTP)

Método Função e Comportamento Exemplo de Uso no Desenvolvimento
GET Solicita um recurso ou representação de dados do servidor (somente leitura) Carregar uma página HTML, carregar imagem ou consultar registros de API
POST Envia novos dados ao servidor para processamento ou criação de recursos Enviar um formulário de cadastro, login ou enviar uma nova postagem
PUT Atualiza ou substitui completamente um recurso existente no servidor Salvar modificações no perfil do usuário, atualizar configurações
DELETE Remove um recurso específico identificado no servidor Excluir uma foto, deletar uma conta de usuário ou registro de banco de dados

Códigos de Status HTTP Comuns

Código Significado / Categoria Exemplo prático de ocorrência
200 OK Requisição bem-sucedida · Sucesso geral Página solicitada carrega normalmente no navegador
301 Redirect Redirecionamento permanente para uma nova URL Usuário acessa site antigo e é levado ao novo endereço automaticamente
404 Not Found O recurso solicitado não pôde ser encontrado no servidor Digitar uma URL incorreta ou acessar página que foi apagada
500 Server Error Erro interno do servidor · Falha na aplicação backend Bug no código PHP/Node.js do servidor ou banco de dados indisponível

Processos de DNS e DHCP nos Bastidores

Passo a Passo da Resolução DNS:
  1. Usuário digita www.senai.br
  2. Computador busca no cache local de DNS
  3. Caso não ache, consulta o servidor DNS recursivo
  4. Recursivo pesquisa a hierarquia: Raiz (.) -> TLD (.br) -> Autoritativo (senai.br)
  5. Servidor retorna o IP e o navegador se conecta
DHCP — Processo DORA (UDP 67/68):
  1. Discover: Cliente envia broadcast procurando servidor
  2. Offer: Servidor oferece um endereço IP disponível
  3. Request: Cliente solicita formalmente o IP oferecido
  4. Acknowledge: Servidor confirma e define o tempo de concessão (lease)

FTP × SFTP — Qual Escolher para Transferência de Arquivos

❌
FTP — File Transfer Protocol
  • Sem criptografia: todas as credenciais (usuário e senha) e dados trafegam em texto puro
  • Vulnerabilidade: interceptações de tráfego na rede (sniffing) expõem arquivos e senhas
  • Conexões separadas: canal de controle na porta 21 e canal de dados na porta 20
  • Problemas de Firewall: o uso de canais distintos dificulta a liberação em firewalls locais
Evite em produção. Use apenas em redes internas isoladas onde a segurança não é um fator crítico.
🛡️
SFTP — SSH File Transfer Protocol
  • Criptografia total: dados e autenticação protegidos por criptografia de chaves do protocolo SSH
  • Segurança robusta: imune a ataques de interceptação de tráfego padrão
  • Única porta: utiliza apenas a porta 22 (SSH) para controle e dados simultaneamente
  • Firewall amigável: liberação simples no firewall por trafegar em uma única porta padronizada
Padrão profissional. Use sempre para transferência de arquivos em servidores web e ambientes de nuvem.
