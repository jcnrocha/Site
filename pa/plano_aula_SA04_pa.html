As Três Etapas para um Ambiente Funcional

📋
Requisitos do Sistema
"Antes de instalar: verificar se o computador tem capacidade mínima para rodar a IDE e o JDK sem travar ou perder produtividade"
  • Processador: dual-core 1.6 GHz ou superior
  • RAM: mínimo 4 GB (recomendado 8 GB ou mais)
  • Disco: ao menos 5 GB livres para IDE + JDK + projetos
  • SO: Windows 10/11 · macOS 10.15+ · Linux 64-bit
  • Permissão de administrador para instalar e configurar variáveis de ambiente
📥
Instalação dos Componentes
"Baixar e instalar cada componente na ordem correta: JDK primeiro, depois IDE, depois extensões — ordem garante que a IDE detecte o Java automaticamente"
  • 1. JDK — Temurin (Adoptium) ou Oracle JDK 21 LTS
  • 2. VS Code — site oficial: code.visualstudio.com
  • 3. Extension Pack for Java — marketplace do VS Code
  • 4. Git — git-scm.com (se ainda não instalado)
  • 5. Verificação: java -version · javac -version
⚙️
Configuração do Ambiente
"Após instalar: configurar o sistema para que qualquer terminal reconheça o Java e a IDE encontre o JDK sem caminhos manuais"
  • JAVA_HOME: aponta para a pasta raiz do JDK
  • PATH: adiciona a pasta %JAVA_HOME%\bin ao PATH do sistema
  • Teste: fechar e reabrir o terminal, rodar javac -version
  • VS Code settings: formatação automática ao salvar, fonte 14px, tema escuro
  • Primeiro programa: criar Hello.java, compilar e executar na IDE

Componentes do Ambiente Completo

☕
JDK 21 LTS
Compilador (javac) + Runtime (java) · Base obrigatória para rodar programas Java
🛠️
VS Code
Editor leve com extensões · Gratuito · Lança projetos Java com o Extension Pack instalado
🔌
Extension Pack for Java
Pacote com 6 extensões: linguagem, depurador, Maven, testes e navegador de projetos Java
🔀
Git
Controle de versão (visto na SA02) · Integrado ao VS Code · Necessário para a IDE exibir o painel Source Control
🌍
Variáveis de Ambiente
JAVA_HOME + PATH · Permitem que qualquer terminal e ferramenta encontrem o JDK automaticamente

Ambiente Configurado × Ambiente sem Configuração

✅
Ambiente Corretamente Configurado
  • java -version no terminal → exibe versão do JDK instalado
  • javac -version → exibe versão do compilador Java
  • VS Code abre arquivo .java com autocomplete e sintaxe colorida
  • Botão "Run" aparece automaticamente acima do método main
  • Variável JAVA_HOME aponta para a pasta correta do JDK
  • Build tools (Maven, Gradle) encontram o JDK sem configuração extra
Objetivo da aula: todos os alunos devem sair com o ambiente validado executando o Hello World
❌
Ambiente sem Configuração
  • 'javac' não é reconhecido → JDK ausente ou PATH não configurado
  • VS Code não oferece autocomplete Java → Extension Pack não instalado
  • Botão "Run" não aparece → JAVA_HOME não definido ou extensão desativada
  • Maven falha com JAVA_HOME not set → variável de ambiente ausente
  • Compilação funciona apenas na pasta do JDK → PATH incompleto
  • Erros silenciosos difíceis de diagnosticar sem conhecer a causa-raiz
Situação comum: instalar o JDK sem configurar as variáveis — o terminal não reconhece os comandos

🎮 Dinâmica Diagnosticar o Ambiente — O Que Está Errado?
🔧 Diagnosticar o Ambiente — O Que Está Errado?
⏱️ Duração: 15 minutos
👥 Agrupamento: grupos de 4 a 5 alunos
🎯 Objetivo: desenvolver habilidade de leitura de mensagens de erro e diagnóstico de problemas de configuração — competência essencial do desenvolvedor desde o primeiro dia
1
Distribuição dos casos (1 min) — Cada grupo recebe 8 fichas com mensagens de erro reais do terminal ou da IDE. Tarefa: identificar (a) o que está faltando ou errado e (b) o que fazer para corrigir
2
Diagnóstico em grupo (7 min) — O grupo discute e registra causa + solução para cada erro. Dica do professor: "Leia a mensagem inteira — ela geralmente diz exatamente o que está faltando"
3
Gabarito e discussão (7 min) — Professor projeta cada erro, coleta as respostas dos grupos e explica a causa-raiz e a solução; destaca que erros de configuração têm sempre uma lógica e nunca são aleatórios
Mensagem de erro / sintomaCausa-raizSolução
'javac' não é reconhecido como comando JDK não instalado ou a pasta bin do JDK não está no PATH do sistema Instalar JDK + adicionar %JAVA_HOME%\bin ao PATH
Build failed: JAVA_HOME not set Variável de ambiente JAVA_HOME não foi definida; Maven e Gradle precisam dela para encontrar o JDK Criar variável JAVA_HOME apontando para a pasta raiz do JDK
VS Code abre o arquivo .java sem autocomplete e sem sintaxe colorida Extension Pack for Java não está instalado; o VS Code trata o arquivo como texto puro Instalar "Extension Pack for Java" no marketplace do VS Code
Botão "Run ▶" não aparece acima do método main JAVA_HOME não definido ou extensão Java Language Support desativada / com erro Definir JAVA_HOME e reiniciar o VS Code; verificar status da extensão
error: release version 21 not supported O projeto está configurado para uma versão de Java diferente da instalada no sistema (ex: projeto exige Java 21, JDK instalado é o 11) Instalar o JDK da versão correta ou ajustar a versão no arquivo de configuração do projeto
java -version funciona, mas javac -version dá erro Apenas o JRE (Java Runtime) foi instalado, mas não o JDK (que inclui o compilador javac) Instalar o JDK completo (não o JRE) — o JDK já inclui o JRE
Terminal reconhece javac na primeira vez mas para de funcionar após reiniciar o computador O PATH foi adicionado apenas para a sessão atual do terminal (variável temporária), e não como variável de ambiente permanente do sistema Definir a variável PATH permanentemente em Variáveis de Ambiente do Sistema no Painel de Controle
VS Code não encontra o Git e o painel Source Control mostra erro Git não está instalado no sistema, ou a pasta do Git não está no PATH Instalar Git (git-scm.com) e verificar se o instalador adicionou ao PATH automaticamente
💡 Reforço pedagógico: saber diagnosticar erros de configuração é tão importante quanto saber programar. Um desenvolvedor que lê mensagens de erro com atenção resolve em 2 minutos o que levaria horas de tentativa e erro. "A mensagem de erro não é inimiga — é o sistema explicando exatamente o que está faltando."
