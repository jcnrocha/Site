As Três Frentes da Aula

🐞
Tipos de Erro
"Saber classificar o erro é o primeiro passo para encontrá-lo — cada tipo tem um sintoma e uma estratégia"
  • Sintaxe: em compilação — falta de ;, chave não fechada · IDE sublinha em vermelho
  • Lógica: executa, mas o resultado é errado — laço infinito, cálculo incorreto
  • Runtime: para abruptamente — NullPointerException, divisão por zero
🔬
Depurador da IDE
"Inspecionar qualquer variável em qualquer momento sem modificar o código — mais preciso, rápido e profissional que o println"
  • Breakpoint pausa · F5 inicia o debug e continua
  • Step Over F10 · Step Into F11 · Step Out Shift+F11
  • Painéis Variables, Watch e Call Stack
  • Breakpoint condicional: pausa só quando i == 537 — essencial em laços longos
📜
Logs e Rastreabilidade
"Em produção o depurador não está disponível — logs são o mecanismo de rastreabilidade do programa em execução"
  • Níveis: DEBUG · INFO · WARN · ERROR
  • Java: java.util.logging.Logger
  • Python: import logging nativo
  • Registrar eventos importantes: login, pedido criado, falha de conexão

Cinco Ferramentas Essenciais do Depurador

🔴
Breakpoint
Marca a linha onde a execução pausa — clicar na margem esquerda · condicional pausa só na iteração problemática
⏭️
Step Over · F10
Executa a linha atual sem entrar dentro das funções chamadas
⤵️
Step Into · F11
Entra dentro da função chamada na linha atual — Step Out (Shift+F11) sai e retorna
👁️
Variables / Watch
Exibe os valores das variáveis no escopo atual · Watch monitora uma expressão específica
🥞
Call Stack
Mostra a pilha de chamadas — qual função chamou qual até o ponto pausado

Depurar com println × Depurar com o Depurador

🖨️
println — o jeito iniciante
  • Modifica o código para inspecionar valores
  • Precisa recompilar e executar a cada nova suspeita
  • Deixa "lixo" no código — prints esquecidos vão para produção
  • Mostra apenas o que foi explicitamente impresso
  • Funciona, mas é lento e impreciso
Aceitável para verificações rápidas — não para investigação séria
🔬
Depurador — o jeito profissional
  • Inspeciona qualquer variável sem modificar o código
  • Pausa exatamente onde interessa — inclusive com condição
  • Navega o fluxo passo a passo: step over, into, out
  • Call Stack mostra o caminho completo até o erro
  • Mais preciso, mais rápido e mais profissional
A ferramenta padrão para investigar qualquer comportamento inesperado

Estratégia Profissional: Reproduzir → Isolar → Breakpoint → Inspecionar → Corrigir → Verificar

ErroSintomaEstratégia de depuração
NullPointerException Programa para com NPE Breakpoint na linha do erro · verificar qual objeto é null no painel Variables
Resultado errado em cálculo Valor incorreto exibido Breakpoint antes do cálculo · inspecionar cada variável passo a passo
Laço infinito Programa trava sem resposta Pausar execução manualmente · verificar condição de saída no painel Variables
Dado não salvo no banco Operação parece funcionar mas dado não persiste Verificar se commit() está sendo chamado · log após cada operação
📜 Níveis de log na prática: DEBUG — valores e fluxo interno para desenvolvimento · INFO — eventos normais (usuário logado, pedido criado) · WARN — situação inesperada mas não crítica (tentativa de login inválida) · ERROR — erro que impede uma operação (falha na conexão com banco, exceção não tratada).

🎮 Dinâmica Caçada ao Bug
🐞 Caçada ao Bug — Depuração de Programas com Erros Intencionais
⏱️ Duração: 15 minutos
👥 Agrupamento: grupos de 4 a 5 alunos
🎯 Objetivo: aplicar a estratégia de depuração para encontrar e corrigir erros em programas com bugs intencionais — desenvolvendo a capacidade de raciocínio analítico e o uso prático das ferramentas de depuração da IDE
1
Distribuição dos programas (1 min) — Cada grupo recebe um programa diferente com um bug intencional. O desafio é: identificar o tipo de erro · usar o depurador para encontrar a causa raiz · corrigir o código · descrever o processo que usaram
2
Depuração em grupo (10 min) — Os grupos usam o depurador da IDE — breakpoints, step over/into e painel de variáveis — para encontrar e corrigir o bug. Registram na ficha: onde colocaram o breakpoint · o que observaram · qual era o erro · como corrigiram
3
Relato e debate (4 min) — Cada grupo relata o bug que encontrou e o processo de depuração. O professor complementa com dicas adicionais de cada caso
ProgramaBug intencionalTipoO que o depurador revela
Calculadora de média Divisão por 2 em vez de 3 para média de 3 notas Lógica Valor da variável divisor é 2 no painel Variables
Busca em vetor Condição do for: i <= lista.length em vez de i < lista.length Runtime ArrayIndexOutOfBoundsException na última iteração
Transferência bancária commit() comentado — dados não são persistidos Lógica Banco retorna 0 linhas afetadas no log
Verificação de senha = em vez de == na comparação Lógica Variável recebe valor em vez de comparar — sempre verdadeiro
💡 Reforço pedagógico: encontrar bugs é uma habilidade que se desenvolve com prática — quanto mais o aluno depurar, mais rápido ficará em identificar padrões de erro. No mercado, desenvolvedores que sabem depurar bem são extremamente valorizados porque reduzem o tempo de resolução de incidentes em produção.
