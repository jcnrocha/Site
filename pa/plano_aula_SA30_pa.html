As Três Frentes da Revisão

🛠️
Ambiente e Ferramentas (SA 01–04)
"Ambiente configurado é pré-requisito de tudo — IDE, Git e GitHub formam a bancada de trabalho do desenvolvedor"
  • Ferramentas do ciclo: IDE · repositório · controle de versão · compilador · gerenciador de dependências
  • Git: init · add · commit · push · pull
  • VS Code: terminal integrado · depurador · extensões · formatação · integração com Git
  • Configuração: requisitos mínimos · PATH · formatOnSave
⚙️
Programação Estruturada (SA 05–07)
"O estruturado é a base de tudo — variáveis, controle, repetição, vetores e funções aparecem em qualquer sistema"
  • Tipos: int, double, String, boolean, char · camelCase
  • Operadores aritméticos, relacionais e lógicos (&&, ||, !)
  • if/else · for · while · do-while
  • Vetores: índice em 0 · .length · funções: void × retorno · depurador
✨
Qualidade Integrada
"A revisão não repete o conteúdo de antes — agora tudo é feito com as boas práticas aprendidas ao longo do ano"
  • Formatação automática (Prettier) em todos os exercícios
  • Nomes descritivos — sem x, n, temp sem contexto
  • Comentar decisões não óbvias — não comentar o óbvio
  • break quando a resposta já foi encontrada · índice sempre com .length

Controle de Versão na Prática — Mensagem de Commit Boa × Ruim

✅
Boas mensagens de commit
  • "adiciona função de cálculo de média"
  • "corrige índice fora do limite no vetor de notas"
  • Descrevem o que foi feito, não o como
  • Qualquer colega entende o histórico sem abrir o código
Fluxo completo: criar pasta → git init → código → git add . → git commit -m "mensagem clara" → git push
❌
Más mensagens de commit
  • "update"
  • "correção"
  • "ajustes finais"
  • Não dizem nada — o histórico vira um mistério
Histórico sem contexto custa caro em qualquer projeto de equipe

Exercícios Práticos de Consolidação

ExercícioEnunciado
1 — Funções e vetores Criar uma função que recebe um vetor de notas e retorna: a maior nota · a menor nota · a média da turma · quantos alunos foram aprovados (nota ≥ 7.0)
2 — Repetição e condicionais Criar um programa que lê números inteiros até o usuário digitar 0 e exibe: a soma total, a quantidade de números positivos e a quantidade de negativos
3 — Repositório Criar repositório no GitHub, commitar os dois exercícios anteriores com mensagens descritivas e verificar o histórico de commits no repositório remoto

🎮 Dinâmica Revisão em Dupla
👥 Revisão em Dupla — Ensino Recíproco
⏱️ Duração: 15 minutos
👥 Agrupamento: duplas
🎯 Objetivo: consolidar os conteúdos de ambiente, ferramentas e programação estruturada por meio de revisão mútua — um aluno explica para o outro, alternando os papéis, desenvolvendo ao mesmo tempo a compreensão do conteúdo e a capacidade de comunicação técnica
1
Distribuição dos tópicos (1 min) — Cada dupla recebe uma ficha com 4 tópicos. O Aluno A explica os 2 primeiros para o Aluno B · o Aluno B explica os 2 últimos para o Aluno A
2
Explicação mútua (10 min) — Cada aluno explica seus tópicos com suas próprias palavras, usando exemplos práticos. O colega que ouve pode fazer perguntas e corrigir se necessário
3
Consolidação coletiva (4 min) — Professor escolhe 2 duplas para compartilhar uma explicação cada com a turma · complementa e corrige se necessário
Tópico da fichaPergunta guia
Diferença Git × GitHub O que é o Git? O que é o GitHub? Por que precisamos dos dois?
Estruturas de repetição Qual a diferença entre for, while e do-while? Quando usar cada um?
Funções com retorno Qual a diferença entre uma função void e uma com retorno? Dê um exemplo de cada
Vetores e índice O que é um vetor? Por que o índice começa em 0? O que acontece se acessar um índice inválido?
💡 Reforço pedagógico: ensinar é a forma mais eficaz de aprender — quando o aluno precisa explicar um conceito com suas próprias palavras, ele identifica exatamente o que sabe e o que ainda não entendeu completamente. Esse método é o ensino recíproco, técnica pedagógica amplamente validada em contextos de educação técnica.
📝 Avaliação Questionário · 10 questões
Parte 1 — Questões Objetivas (5 questões)
  • Nível alto · 4 alternativas: A) B) C) D)
  • Gabarito alternado, sem repetição consecutiva de letras
  • Temas: papel de cada ferramenta do ciclo, sequência correta dos comandos Git, qual estrutura de repetição usar em cada cenário, comportamento de vetores e índices, diferença entre função void e com retorno
Parte 2 — Questões Discursivas (5 questões)
  • Cada questão possui título explicativo antes do enunciado
  • Sem linhas — apenas espaço em branco com R:
  • Temas: escrever a função do exercício 1 (vetor de notas), escrever o programa do exercício 2 (leitura até zero), descrever o fluxo completo do Git com mensagens de commit boas, explicar por que nomes descritivos são qualidade, identificar e corrigir um erro de índice em um trecho dado
Gabarito
  • Em página separada — uso exclusivo do professor
  • Objetivas: letra correta + justificativa completa
  • Discursivas: resposta esperada com critérios de correção e exemplos de código válidos
