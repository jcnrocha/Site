Três Fundamentos da Programação Estruturada

📦
Variáveis e Tipos
"Uma variável é um espaço nomeado na memória que armazena um valor de um tipo específico — o tipo define quanto espaço ocupar e quais operações são permitidas"
  • int → inteiros: 0, 1, -42, 1000
  • double → decimais: 3.14, -0.5, 99.99
  • boolean → verdadeiro/falso: true, false
  • char → um único caractere: 'A', '9', '@'
  • String → sequência de texto: "Olá, Mundo!"
🔢
Operadores
"Operadores permitem realizar cálculos, comparar valores e combinar condições — são a linguagem das expressões dentro do código"
  • Aritméticos: + - * / % (resto da divisão)
  • Relacionais: == != > < >= <=
  • Lógicos: && (E) · || (OU) · ! (NÃO)
  • Atribuição: = += -= *= /=
  • Atenção: = atribui · == compara
🔀
Estrutura Condicional
"O if/else é a estrutura de seleção — permite que o programa tome decisões: se uma condição for verdadeira, execute um bloco; senão, execute outro"
  • if (condição) { ... } — bloco executado se condição for true
  • else if (outra) { ... } — alternativa, testada somente se o if anterior foi false
  • else { ... } — bloco padrão quando nenhuma condição anterior foi verdadeira
  • Condições podem usar operadores relacionais e lógicos
  • Blocos podem ser aninhados (if dentro de if)

Tipos de Dados em Java

🔢
int
Números inteiros sem casas decimais · Ex: int idade = 17;
🔣
double
Números com casas decimais · Ex: double preco = 49.90;
✅
boolean
Verdadeiro ou falso · Ex: boolean aprovado = true;
🔤
char
Um único caractere (aspas simples) · Ex: char letra = 'A';
💬
String
Texto (aspas duplas) · Ex: String nome = "Ana";

Boas Práticas de Nomenclatura × Nomenclatura Inadequada

✅
Boas Práticas de Nomenclatura
  • camelCase para variáveis e métodos: idadeUsuario, calcularSalario()
  • PascalCase para classes: ContaBancaria, FolhaPagamento
  • SNAKE_CASE_MAIÚSCULO para constantes: TAXA_JUROS
  • Nomes descritivos: totalVendas em vez de tv
  • Sem abreviações obscuras: quantidadeProdutos não qtdPrd
  • Idioma consistente: todo o código em português OU todo em inglês
Adote desde o primeiro programa — refatorar nomes depois custa muito mais tempo do que nomear bem desde o início
⚠️
Nomenclatura Inadequada
  • Nomes sem significado: x, y, aux, temp em variáveis de negócio
  • Mistura de idiomas: valorPrice, nomeEmployee
  • Abreviações confusas: qtdPrd, vl, nr
  • snake_case em variáveis Java: nome_usuario (não é convenção Java)
  • Nomes enganosos: variável chamada lista guardando um único número
  • Nomes que exigem comentário para entender o que fazem
Código mal nomeado é código difícil de manter — em equipe, ninguém entende; no futuro, nem o próprio autor

1
Distribuição das situações (1 min) — Cada grupo recebe 8 fichas com situações do mundo real. Para cada uma, o grupo deve escrever a declaração completa em Java: tipo + nome (camelCase) + valor inicial
2
Programação em grupo (8 min) — O grupo escreve as 8 declarações em papel ou na IDE. Um aluno digita enquanto os outros orientam. Grupos que terminam antes escrevem um bloco if/else usando duas das variáveis criadas
3
Apresentação e gabarito (6 min) — Professor projeta cada situação, solicita a resposta de um grupo e compara com o gabarito; discute variações válidas e erros comuns (tipo errado, nome inadequado, uso de = em vez de ==)
SituaçãoDeclaração correta em JavaPor que esse tipo?
Armazenar a idade de um usuário cadastrado no sistema int idadeUsuario = 25; Idade é sempre inteira — não existe "17,5 anos" em cadastros
Armazenar o preço de um produto em um e-commerce double precoProduto = 49.90; Preços têm centavos — precisam de casas decimais

Armazenar a primeira letra do turno (M = manhã, T = tarde, N = noite) char turno = 'M'; Um único caractere → char com aspas simples
Armazenar o nome completo de um cliente String nomeCliente = "Maria Silva"; Texto com múltiplos caracteres → String com aspas duplas
Armazenar a quantidade de itens no carrinho de compras int quantidadeItens = 3; Contagem de itens é sempre inteira — nunca "2,5 produtos"
