As Três Frentes da Revisão

🏗️
Orientação a Objetos (SA 08–12)
"POO organiza o sistema em entidades com dados e comportamento — o coração do Projeto Final"
  • Classe é o molde · objeto é a instância · new instancia · construtor inicializa
  • private protege · getter lê · setter valida · this referencia
  • extends herda · super acessa o pai · @Override redefine
  • Polimorfismo: referência pai → objeto filho · abstratas e interfaces
🗄️
Banco de Dados (SA 13–15)
"O banco persiste — conexão, CRUD completo e transações revisados em conjunto com a POO"
  • Conexão: driver · host, porta, banco, usuário, senha
  • SELECT: executeQuery() · while(rs.next())
  • INSERT/UPDATE/DELETE: executeUpdate() · Prepared Statement · WHERE obrigatório
  • Transações: setAutoCommit(false) · commit · rollback · finally fecha
🧑‍💼
Sistema Prático Integrado
"Cadastro de Funcionários com BD — POO e CRUD juntos em um único sistema coeso, como será no Projeto Final"
  • Hierarquia Pessoa (abstrata) → Funcionario
  • Classe de acesso ao banco FuncionarioBD
  • Polimorfismo ao percorrer a lista chamando exibirDados()
  • Refatoração com DRY, encapsulamento, documentação e commits

Sistema de Consolidação — Cadastro de Funcionários com BD

ClasseEstrutura
Pessoa (abstrata) nome e cpf private · construtor(nome, cpf) · exibirDados() abstrato
Funcionario extends Pessoa cargo e salario private · construtor(nome, cpf, cargo, salario) · exibirDados() @Override · calcularBonus() retorna 10% do salário
FuncionarioBD (acesso ao banco) inserir(Funcionario f) · buscarTodos() · buscarPorNome(String) · atualizar(Funcionario f) · remover(int id)
Sistema principal Cria objetos com construtor → insere no banco → busca todos e percorre com for chamando exibirDados() (polimorfismo) → atualiza salário → remove pelo ID

Cinco Erros Frequentes a Revisar e Corrigir

💥
NullPointer
Exception
Objeto não instanciado com new — verificar a criação antes do uso
🎯
UPDATE sem WHERE
Esquecimento da cláusula — sempre incluir WHERE com ID ou condição específica
🔓
private violado
Atributo acessado diretamente — usar getter/setter correspondente
🚰
Conexão aberta
Vazamento de recurso — sempre fechar no finally ou try-with-resources
🔄
@Override errado
Assinatura diferente do método pai — conferir nome, parâmetros e retorno idênticos
✨ Refatoração integrada — qualidade no sistema de consolidação: DRY — método de abertura de conexão centralizado em uma classe ConexaoDB · encapsulamento — nenhum atributo acessado diretamente fora da classe · documentação — Javadoc/docstring em todos os métodos públicos · tratamento de erro — try/catch/finally em todos os métodos de BD · commit de cada etapa com mensagem descritiva no repositório.

🔗 Liga o Conceito ao Código — Leitura Técnica de POO e CRUD
⏱️ Duração: 15 minutos
👥 Agrupamento: grupos de 4 a 5 alunos
🎯 Objetivo: consolidar POO e CRUD por meio da identificação de conceitos em trechos de código reais — desenvolvendo a leitura técnica precisa que será necessária no Projeto Final e em revisões de código no mercado de trabalho
1
Apresentação do trecho (1 min por rodada) — O professor projeta um trecho de código. O grupo deve identificar: qual conceito está sendo demonstrado · se está correto ou tem problema · e o que melhoraria
2
Análise em grupo (8 min) — Os grupos analisam cada trecho e preenchem a ficha com: conceito identificado, avaliação (correto/problema) e sugestão de melhoria quando necessário
3
Debate coletivo (6 min) — O professor revela as respostas e discute as sugestões de melhoria apresentadas pelos grupos
TrechoConceitoAvaliação
Pessoa p = new Funcionario("Ana", "123", "Dev", 5000) Polimorfismo — referência pai para objeto filho Correto
public String nome; dentro de uma classe Encapsulamento violado — atributo público Problema — deve ser private com getter/setter
UPDATE funcionarios SET salario = 6000 sem WHERE CRUD — UPDATE sem condição Problema crítico — atualiza todos os registros
@Override public void exibirDados(String extra) diferente do pai Herança — tentativa de @Override com assinatura diferente Problema — não é Override válido, é sobrecarga
finally { conexao.close(); } após try/catch Boas práticas de CRUD — fechamento de conexão Correto
Método com 60 linhas fazendo conexão + CRUD + exibição DRY e responsabilidade única Problema — dividir em métodos separados
💡 Reforço pedagógico: identificar conceitos em código alheio é uma habilidade cotidiana do desenvolvedor — em revisões de código (code review), em manutenção de sistemas legados e em entrevistas técnicas. Quem consegue ler código com precisão técnica e apontar melhorias com clareza é um profissional muito valorizado em qualquer equipe.
