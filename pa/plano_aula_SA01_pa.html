Do Ambiente de Desenvolvimento ao Aplicativo Final

🛠️
Ambiente de Desenvolvimento
"O conjunto de ferramentas que o desenvolvedor usa para escrever, testar e versionar código"
  • IDE: editor inteligente com debug e extensões
  • Compilador/Interpretador: executa o código
  • Repositório Git: controla versões do projeto
  • Terminal integrado: executa comandos diretos
  • Ex: VS Code, IntelliJ IDEA, Eclipse
💻
Código-Fonte
"As instruções escritas pelo desenvolvedor para resolver um problema ou automatizar uma tarefa"
  • Paradigma estruturado: lógica sequencial
  • Paradigma OO: classes, objetos e herança
  • Conexão com banco de dados: persistência de dados
  • Técnicas: formatação, testes e documentação
  • Ex: Python, Java, PHP, JavaScript
📱
Aplicativo Final
"O produto final — entregue ao usuário — resultado da junção do ambiente, do código e dos dados"
  • Aplicativo web: sistemas e portais no navegador
  • Aplicativo desktop: software instalado
  • Aplicativo mobile: Android e iOS
  • API: serviço consumido por outros sistemas
  • Ex: sistema de cadastro, app de vendas

Por Que Aprender Programação de Aplicativos?

💼
Empregabilidade
Mercado aquecido e crescente para desenvolvedores — uma das carreiras mais demandadas em TI
🔨
Criação
Transformar ideias em produtos reais que resolvem problemas de pessoas e organizações
🤖
Automação
Escrever código que executa tarefas repetitivas com precisão, velocidade e sem erros humanos
🌐
Integração
Conectar sistemas, APIs e bancos de dados para criar soluções completas e integradas
🚀
Inovação
Criar novas soluções que impactam o mercado — startups, produtos digitais e sistemas modernos

IDE Profissional × Editor de Texto Simples

🛠️
IDE Profissional (VS Code, IntelliJ…)
  • Autocomplete inteligente e sugestões de código em tempo real
  • Depurador integrado: breakpoints, inspeção de variáveis
  • Terminal embutido: executa comandos sem sair do editor
  • Controle de versão Git integrado com interface visual
  • Extensões: suporte a múltiplas linguagens e frameworks
  • Realce de sintaxe avançado e detecção de erros em tempo real
Use sempre: o ambiente de desenvolvimento profissional aumenta produtividade e reduz erros
📝
Editor de Texto Simples (Bloco de Notas…)
  • Sem autocomplete — o desenvolvedor precisa lembrar tudo
  • Sem depurador — encontrar erros é manual e demorado
  • Sem terminal — é necessário abrir outra janela para executar
  • Sem controle de versão — risco de perder código ou sobrescrever
  • Sem extensões — suporte apenas ao arquivo aberto
  • Erros de sintaxe só aparecem ao tentar executar o código
Evite para projetos reais: adequado apenas para anotações rápidas, não para desenvolvimento

🎮 Dinâmica Já Vi Isso Antes?
🛠️ Já Vi Isso Antes?
⏱️ Duração: 15 minutos
👥 Agrupamento: grupos de 4 a 5 alunos
🎯 Objetivo: levantar o conhecimento prévio da turma sobre ferramentas de desenvolvimento e contextualizar a jornada do ano
1
Distribuição das ferramentas (1 min) — Cada grupo recebe 6 cartões (ou vê os slides projetados) com descrições de uso de ferramentas de desenvolvimento. O desafio: "Já vi isso antes? O que é e para que serve essa ferramenta?"
2
Discussão em grupo (7 min) — O grupo decide: JÁ VI ou NUNCA VI, e tenta nomear a ferramenta e descrever sua função no ciclo de desenvolvimento de um aplicativo
3
Compartilhamento e gabarito (7 min) — Cada grupo apresenta uma ferramenta; professor revela o nome correto, explica a função e conecta ao bloco da matriz onde ela será aprofundada
Descrição de uso da ferramentaFerramentaBloco da UC
"Escrevo meu código aqui, com destaque de cores, sugestões automáticas e clico F5 para executar" IDE B2 — Ambiente
"Quando termino uma funcionalidade, faço um commit e envio para o servidor remoto com git push" Git / GitHub B2 — Repositório
"Defino uma classe Produto com atributos nome e preço, depois crio objetos a partir dela" OO / Linguagem B4 — OO
"Executo um INSERT INTO na minha aplicação para salvar o cadastro do usuário no banco" BD / SQL B5 — Banco de Dados
"Coloco um breakpoint na linha 42, executo em modo debug e inspeciono o valor da variável" Depurador B6 — Depuração
"Escrevo um método de teste que verifica se o cálculo do desconto retorna o valor esperado" Teste Unitário B6 — Testes
"Preencho os 9 blocos do Canvas do meu aplicativo: proposta de valor, clientes, canais e receita" Canvas B10 — Modelagem
"Não compartilho senha de cliente nem dado pessoal de usuário — sigilo é princípio inegociável" Ética em TI B7 — Ética
💡 Reforço pedagógico: o objetivo não é que o aluno já conheça todas as ferramentas, mas perceber que elas formam um ecossistema integrado. "Cada bloco da UC que você estudar este ano vai preencher um espaço nesse mapa de ferramentas." Reforçar que programar bem vai muito além de saber a sintaxe — é sobre ambiente, qualidade, ética e entrega.

  • Cada questão possui título explicativo antes do enunciado
  • Sem linhas — apenas espaço em branco com R:
  • Temas: situações práticas do desenvolvedor, vantagens da IDE, fluxo ambiente → produto, papel da ética em TI, expectativas do aluno para o ano
Gabarito
  • Em página separada — uso exclusivo do professor
  • Objetivas: letra correta + justificativa completa
  • Discursivas: resposta esperada com critérios de correção
