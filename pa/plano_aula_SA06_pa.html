Três Pilares do Programa Estruturado

🔁
Estruturas de Repetição
"Permitem executar um bloco de código múltiplas vezes sem copiar e colar — base de qualquer algoritmo que processa coleções de dados"
  • for(init; cond; inc) — iterações com contador definido
  • while(condição) — repete enquanto condição for verdadeira
  • do { } while(cond) — executa ao menos uma vez antes de verificar
  • for(tipo x : array) — for-each: percorre coleção sem índice
  • break sai do laço · continue pula para próxima iteração
📋
Vetores (Arrays)
"Um vetor é uma sequência de variáveis do mesmo tipo agrupadas sob um único nome — acesso por índice numérico, sempre começando em 0"
  • Declaração: int[] notas = new int[5];
  • Inicialização: int[] n = {7, 8, 9, 6, 10};
  • Acesso: notas[0] = primeiro · notas[4] = último (de 5)
  • Tamanho: notas.length retorna o número de elementos
  • Índice fora do range → ArrayIndexOutOfBoundsException
🧩
Funções e Métodos
"Funções dividem o código em blocos reutilizáveis com nome — cada função faz uma coisa bem feita, e pode ser chamada de qualquer lugar do programa"
  • Sem retorno: static void imprimirMensagem()
  • Com retorno: static int somar(int a, int b)
  • Parâmetros: valores passados para a função usar
  • return devolve um valor ao chamador e encerra a função
  • Princípio: uma função = uma responsabilidade

Quando Usar Cada Estrutura de Repetição

🔢
for
Número de iterações conhecido · Percorrer vetores com índice · Tabuadas e contagens
⏳
while
Repete enquanto condição for verdadeira · Número de iterações desconhecido · Entrada com sentinela
🔄
do-while
Executa ao menos uma vez antes de verificar a condição · Ideal para menus e validação de entrada
📦
for-each
Percorre coleções sem índice explícito · Mais legível · Não permite modificar o array durante o percurso
✂️
break / continue
break encerra o laço imediatamente · continue pula para a próxima iteração

for × while — Como Escolher

🔢
for — iterações definidas
  • Número de repetições conhecido antes de começar
  • Percorrer vetor: for(int i=0; i<v.length; i++)
  • Contadores crescentes ou decrescentes: 1 a 10, 10 a 0
  • Inicialização, condição e incremento ficam na mesma linha — mais compacto e visual
  • Variável i acessível dentro do bloco — útil quando precisa do índice
Use quando: "sei exatamente quantas vezes quero repetir"
⏳
while — condição aberta
  • Número de repetições desconhecido — depende de evento externo
  • Ler entrada até digitar 0: while(valor != 0)
  • Esperar condição mudar: aguardar conexão, arquivo pronto
  • Condição pode ser complexa com operadores lógicos
  • Atenção: se a condição nunca mudar → loop infinito
Use quando: "não sei quantas vezes — repito até algo acontecer"

🔁 Qual Laço Usar? — Escolha e Escreva o Cabeçalho
⏱️ Duração: 15 minutos
👥 Agrupamento: grupos de 4 a 5 alunos
🎯 Objetivo: desenvolver o critério de seleção de estrutura de repetição — habilidade que elimina bugs e melhora a legibilidade do código desde o início
1
Distribuição dos cenários (1 min) — Cada grupo recebe 8 fichas com situações de problema real. Para cada uma: (a) identificar o laço mais adequado e (b) escrever o cabeçalho completo em Java

3
Gabarito e debate (6 min) — Professor projeta cada cenário, ouve as respostas e discute: há sempre uma escolha mais idiomática em Java; aceita respostas alternativas válidas desde que justificadas
CenárioLaço indicadoCabeçalho em Java
Percorrer todas as 30 notas de uma turma e imprimir cada uma for for (int i = 0; i < notas.length; i++)
Continuar pedindo a senha até o usuário digitar a senha correta while while (!senha.equals(senhaCorreta))

Imprimir a tabuada do 7 de 1 a 10 for for (int i = 1; i <= 10; i++)
Ler números inteiros do usuário e somar até ele digitar 0 (valor sentinela) while while (numero != 0)
Fazer uma contagem regressiva de 10 até 1 e imprimir "Lançamento!" for for (int i = 10; i >= 1; i--)

💡 Reforço pedagógico: a escolha da estrutura de repetição comunica intenção. Usar for diz "sei quantas vezes"; usar while diz "não sei — depende de uma condição". Código que escolhe bem as estruturas se lê como uma descrição do problema, não como uma sequência de instruções mecânicas. Atenção ao loop infinito: se usar while, sempre garantir que a condição eventualmente se tornará falsa.
