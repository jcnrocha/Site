As Três Frentes do Briefing

🚀
O Que É o Projeto Final
"Um aplicativo funcional com banco de dados, desenvolvido em grupo — tudo o que foi aprendido no ano converge aqui"
  • Integra todos os conteúdos da UC em um produto real e demonstrável
  • Avaliado em duas dimensões: produto (o sistema funcionando) e processo (como foi desenvolvido)
  • Entregável final: sistema rodando + Canvas + apresentação oral + repositório no GitHub
📋
Requisitos Obrigatórios
"Os requisitos garantem que todo grupo demonstre o domínio dos blocos centrais da UC"
  • Linguagem da UC com paradigma orientado a objetos
  • Ao menos 3 classes com herança e encapsulamento
  • CRUD completo com Prepared Statement
  • Ao menos 5 testes unitários · Javadoc/docstring nos públicos
  • Repositório no GitHub + Canvas + pitch de 5 minutos
👥
Organização dos Grupos
"Grupos de 3 a 4 alunos — trabalho individual rastreável pelos commits de cada membro"
  • Definidos nesta SA: tema · nome do sistema · responsável por módulo
  • Repositório criado no GitHub ainda nesta aula
  • Todos os membros devem ter commits no repositório
  • Tema próprio é permitido — sujeito à aprovação de viabilidade pelo professor

🎮 Dinâmica Integrada à SA
🚀 Esta SA é a própria dinâmica

Esta SA não possui dinâmica separada — a própria SA é uma grande dinâmica prática de planejamento e organização do Projeto Final. A interação entre os grupos, a escolha do tema, o preenchimento do Canvas e a criação do repositório constituem a atividade central desta SA.

💡 Orientação ao professor: circular entre os grupos durante o Canvas inicial e o planejamento, fazendo perguntas desafiadoras ("esse escopo cabe em 3 SAs de desenvolvimento?") — o sucesso do Projeto Final depende de um escopo viável definido hoje.
📝 Avaliação Entregável do grupo
Entregáveis desta SA — sem questionário individual
  • Canvas inicial preenchido (os 9 blocos com pelo menos os 4 blocos principais completos)
  • Repositório criado no GitHub com README descrevendo o sistema, os integrantes e o escopo
  • Divisão de responsabilidades documentada
Avaliação formativa
  • Professor registra observações sobre organização, clareza do escopo e viabilidade técnica de cada grupo
  • A avaliação somativa do projeto será aplicada na SA 38 (apresentação) e na SA 39 (avaliação do 3º Trimestre)
