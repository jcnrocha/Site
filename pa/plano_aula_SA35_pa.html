As Três Frentes do Desenvolvimento de Hoje

🗄️
CRUD Completo
"As quatro operações funcionando com Prepared Statement — checklist completo antes de avançar"
  • INSERT com Prepared Statement e commit
  • SELECT sem filtro e com WHERE (por ID ou campo relevante)
  • UPDATE com WHERE obrigatório e verificação de rowcount
  • DELETE com WHERE obrigatório e confirmação de exclusão
  • Transação onde duas operações dependem uma da outra
🛡️
Erros e Documentação
"try/catch em toda operação de banco com mensagens informativas — e Javadoc/docstring em todos os públicos"
  • Padrão mínimo: try-with-resources + catch (SQLException e) com mensagem clara
  • Verificar linhasAfetadas == 0 e informar "ID não encontrado"
  • Javadoc com @param e @return em cada método público
  • Cada método com sua própria descrição — não copiar de outro
🧪
Testes Unitários
"Testar o modelo e as regras de negócio — sem dependência de banco real"
  • Testar: getters, setters com validação, construtores e cálculos
  • Não testar DAO diretamente — usa banco real (mock é avançado)
  • Exemplos: naoDeveAceitarEmailSemArroba() · deveLancarExcecaoQuandoNomeForVazio()
  • Ao menos um cenário de borda ou erro por método testado

Erros Frequentes Nesta Fase — Orientações Preventivas

Erro frequenteComo evitar
UPDATE sem WHERE atualiza tudo Sempre verificar que o WHERE está presente antes de executar
DELETE sem WHERE remove tudo Mesma orientação — WHERE é obrigatório
Testes só do cenário feliz Incluir ao menos um cenário de borda ou erro em cada método
Documentação copiada e colada Cada método tem sua própria descrição — não copiar de outro método
Commit com código quebrado Commitar apenas código que compila — nunca subir código quebrado
✨ Orientações de qualidade para esta SA: revisar o código da SA 34 antes de continuar — corrigir problemas identificados pelo professor · aplicar DRY: abertura de conexão duplicada em DAOs deve ser extraída para ConexaoDB · verificar que todos os atributos estão private · commitar ao terminar cada método que funciona — não acumular para o final. Padrão de UPDATE trabalhado em aula: UPDATE clientes SET nome = ?, email = ? WHERE id = ? com verificação de linhasAfetadas.
