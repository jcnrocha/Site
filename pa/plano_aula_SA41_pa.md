Projeto Final — Encerramento e Exposição

## Objetivos de Aprendizagem

Ao final da SA, o aluno será capaz de:

- Apresentar o Canvas completo do projeto com coerência entre modelo de negócio e sistema desenvolvido
- Expor o sistema funcionando para uma audiência ampliada de forma profissional
- Refletir sobre o processo de desenvolvimento vivenciado ao longo do projeto
- Aplicar autoavaliação crítica identificando pontos de evolução e oportunidades de melhoria
- Consolidar a experiência do Projeto Final como referência para os próximos projetos da formação

**Capacidades da matriz:** Integração de todas as capacidades da UC · Canvas · Capacidades socioemocionais: reflexão, ética, organização e profissionalismo

---

## Conteúdo

- Contextualização: *"As avaliações estão feitas — esta SA é a celebração do que foi construído. Não é mais uma prova: é a exposição do produto final e a reflexão sobre a jornada"*

- **Formato da exposição:**
  - Cada grupo monta uma estação de demonstração com o sistema rodando
  - Os demais grupos e o professor circulam pelas estações como visitantes
  - Cada membro do grupo deve ser capaz de demonstrar e explicar o sistema individualmente
  - Canvas impresso ou projetado visível na estação

- **O que deve estar disponível em cada estação:**

  | Item | Descrição |
  |------|-----------|
  | Sistema rodando | Aplicativo aberto e funcional com banco conectado e dados reais |
  | Canvas impresso ou projetado | Todos os 9 blocos preenchidos e visíveis |
  | README impresso ou projetado | Descrição do sistema, tecnologias e como rodar |
  | Histórico de commits | Repositório aberto no navegador mostrando o histórico |
  | Resumo técnico | Folha com: estrutura de classes · quantidade de testes · cobertura |

- **Roteiro da demonstração na estação — 3 minutos por visitante:**

  | Momento | O que apresentar |
  |---------|-----------------|
  | Proposta de valor | Uma frase: qual problema resolve e para quem |
  | Demonstração | Mostrar o fluxo principal: cadastrar → buscar → atualizar → remover |
  | Canvas | Apontar os dois blocos mais importantes para o sistema |
  | Destaque técnico | Uma decisão técnica que o grupo se orgulha |
  | Pergunta aberta | *"Tem alguma dúvida sobre o sistema ou sobre o desenvolvimento?"* |

- **Autoavaliação do Projeto Final — reflexão individual:**
  - Cada aluno preenche individualmente a ficha de autoavaliação ao final da SA
  - Objetivo: desenvolver consciência sobre o próprio processo de aprendizagem

  **Perguntas da autoavaliação:**
  1. Qual foi a maior dificuldade técnica que você enfrentou no projeto e como resolveu?
  2. Qual conceito da UC você sente que dominou melhor ao longo do ano?
  3. Qual conceito você ainda sente que precisa desenvolver mais?
  4. O que faria diferente se fosse começar o projeto do zero hoje?
  5. Como avalia sua contribuição para o grupo? (de 1 a 5 com justificativa)

- **Reflexão coletiva — encerramento do projeto:**
  - Após a exposição, professor conduz uma conversa rápida com a turma inteira
  - Perguntas para a reflexão coletiva:
    - *"Qual sistema da turma vocês gostariam de ter desenvolvido? Por quê?"*
    - *"Qual foi o bug mais criativo que alguém da turma enfrentou?"*
    - *"Se vocês fossem contratar um desenvolvedor, qual habilidade vista neste projeto seria mais importante?"*

## Organização do Tempo

| Etapa | Atividade |
|-------|-----------|
| Abertura | Chamada · orientações da exposição: formato, tempo por estação, postura como apresentador e como visitante |
| Montagem das estações | Cada grupo organiza sua estação: sistema rodando, Canvas visível, README disponível |
| Exposição | Professor e grupos circulam pelas estações · cada visita dura 3 minutos · professor faz perguntas técnicas e de negócio em cada estação |
| Autoavaliação | Alunos preenchem a ficha de autoavaliação individual em silêncio |
| Reflexão coletiva | Professor conduz conversa de encerramento do projeto com a turma |
| Fechamento | Professor encerra formalmente o Projeto Final · orienta sobre a SA 42 (última SA do ano) |
