As Três Frentes da Aula

🧭
Do Problema ao Valor
"O Canvas não começa pelo produto — começa pelo cliente e pelo problema que ele tem"
  • 1. Quem é o cliente? (Segmento)
  • 2. Qual problema ele tem? (base da Proposta de Valor)
  • 3. Como o sistema resolve? (Proposta de Valor)
  • 4. Como chega até ele? (Canais)
  • 5. Como se sustenta? (Fontes de Receita)
👥
Segmentos de Clientes
"Sistemas podem atender múltiplos segmentos com necessidades diferentes — cada um precisa ser descrito"
  • B2C: serve o consumidor final (app de delivery)
  • B2B: serve outras empresas (gestão para clínicas)
  • B2B2C: serve empresas que servem consumidores (plataforma de e-commerce)
  • Multilateral: dois ou mais grupos interdependentes (marketplace: vendedores e compradores)
🔧
Canvas → Decisões Técnicas
"Cada escolha de arquitetura tem uma razão de negócio por trás — o Canvas evita decisões técnicas desconectadas"
  • Segmento jovem mobile-first → app mobile responsivo, não desktop
  • Receita freemium → controle de plano e bloqueio de funcionalidades premium
  • Parceria via Open Finance → integração com APIs externas
  • Relacionamento self-service → interface intuitiva sem treinamento

Cinco Modelos de Receita para Software (+ Licença)

🔁
SaaS (Assinatura)
Pagamento recorrente pelo acesso — Spotify, Netflix, GitHub Pro
🆓
Freemium
Versão gratuita limitada · paga com recursos completos — Trello, Notion, Canva
💳
Por Transação
Cobrança sobre cada operação — iFood (comissão por pedido), Mercado Pago
📣
Publicidade
Receita de anunciantes que pagam para exibir anúncios — Google, YouTube gratuito
🏪
Marketplace
Comissão sobre transações entre compradores e vendedores — Airbnb, Uber, Etsy · há ainda a Licença (pagamento único)

Funcionalidade × Proposta de Valor — o Bloco Mais Importante

🧱
Funcionalidade — o que o sistema faz
  • "Cadastro de tarefas"
  • "Relatório de vendas"
  • "Sistema de agendamento"
  • "Controle de estoque"
  • Descreve o recurso, não o motivo de usar
Funcionalidade não responde: "por que o cliente usaria ESTE sistema?"
💎
Proposta de Valor — o benefício real
  • "Nunca mais esquecer um compromisso importante"
  • "Tomar decisões de negócio com dados reais em segundos"
  • "Eliminar ligações telefônicas e reduzir faltas em consultas"
  • "Nunca perder uma venda por falta de produto"
  • Específica, clara e orientada ao benefício
Responde: "por que o cliente usaria este sistema e não outro?"

🎮 Dinâmica Canvas ao Vivo
🎨 Canvas ao Vivo — Modelando um Sistema do Próprio Grupo
⏱️ Duração: 15 minutos
👥 Agrupamento: grupos de 4 a 5 alunos
🎯 Objetivo: criar um Canvas completo para um sistema de software proposto pelo próprio grupo — desenvolvendo o raciocínio integrado de negócio e tecnologia que será usado diretamente no Projeto Final das SAs 33 a 38
1
Escolha do sistema (1 min) — Cada grupo escolhe um tipo de sistema simples para modelar. O professor sugere opções caso o grupo não tenha ideia própria
2
Preenchimento do Canvas (9 min) — Os grupos preenchem os 9 blocos do Canvas para o sistema escolhido, começando sempre pelo Segmento de Clientes e pela Proposta de Valor
3
Apresentação rápida (5 min) — Cada grupo apresenta em 1 minuto: quem é o cliente, qual o problema resolvido e como o sistema gera receita. Professor faz uma pergunta desafiadora para cada grupo
Sistema sugeridoPúblico sugerido
App de agendamento de serviços (cabeleireiro, dentista)B2C e B2B
Sistema de controle de estoque para pequeno comércioB2B
Plataforma de caronas entre alunos de faculdadeB2C multilateral

💡 Reforço pedagógico: este exercício é um ensaio do Projeto Final — os grupos que se saírem bem aqui terão muito mais facilidade nas SAs 33 a 38. Encorajar os alunos a escolher um sistema que realmente gostariam de construir — a motivação pelo tema faz diferença na qualidade do projeto entregue.
📝 Avaliação Questionário · 10 questões
Parte 1 — Questões Objetivas (5 questões)
  • Nível alto · 4 alternativas: A) B) C) D)
  • Gabarito alternado, sem repetição consecutiva de letras
  • Temas: a sequência correta de raciocínio do Canvas, identificar proposta de valor × funcionalidade, classificar um sistema como B2C/B2B/multilateral, escolher o modelo de receita adequado a um cenário, qual decisão técnica decorre de um bloco do Canvas
Parte 2 — Questões Discursivas (5 questões)
  • Cada questão possui título explicativo antes do enunciado
  • Sem linhas — apenas espaço em branco com R:
  • Temas: transformar três funcionalidades em propostas de valor, preencher os blocos principais do Canvas para um sistema dado, comparar dois modelos de receita para o mesmo aplicativo com justificativa, explicar como o segmento de clientes influencia decisões de interface, escrever o pitch de proposta de valor usando a fórmula apresentada
Gabarito
  • Em página separada — uso exclusivo do professor
  • Objetivas: letra correta + justificativa completa
  • Discursivas: resposta esperada com critérios de correção
