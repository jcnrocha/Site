## SA 01 — 1º Trimestre

---

## Tema

Apresentação da UC e Introdução ao Ambiente de Desenvolvimento

---

## Objetivos de Aprendizagem

Ao final da SA, o aluno será capaz de:

- Compreender o objetivo da UC Programação de Aplicativos e sua relação com o perfil profissional do Técnico em Desenvolvimento de Sistemas
- Identificar as principais ferramentas utilizadas no desenvolvimento de aplicativos e suas funções no ciclo de desenvolvimento
- Reconhecer o que é um repositório de código e qual o papel do controle de versão no trabalho em equipe
- Relacionar o conceito de programação de aplicativos com situações reais do mercado de trabalho em TI
- Descrever, com suas próprias palavras, o que um técnico em desenvolvimento de sistemas faz no dia a dia profissional

**Capacidades da matriz:** Reconhecer ferramentas para o desenvolvimento de atividades (repositório, controle de versão) · Reconhecer especificações técnicas e paradigmas de linguagem de programação

## Conteúdo

- **Apresentação da UC:**
  - Objetivo geral da UC: criar aplicativos por meio de linguagem de programação com padrão de qualidade, robustez, integridade e segurança
  - Carga horária: 133h · 42 SAs · Quartas-feiras · 50 min/SA
  - Como a UC se conecta com as demais do curso Técnico em Desenvolvimento de Sistemas
  - Metodologia: situações de aprendizagem práticas, questionários, projeto final com Canvas

- **Perfil profissional do Técnico em Desenvolvimento de Sistemas:**
  - Função: programar sistemas computacionais atendendo normas e padrões de qualidade
  - O que o técnico faz: codifica, testa, documenta e mantém sistemas
  - Onde atua: empresas de software, departamentos de TI, startups, trabalho remoto
  - Relação entre teoria e prática: por que aprender a programar bem desde o início importa

- **O que é programação de aplicativos:**
  - Definição: processo de criação de um programa ou sistema para resolver um problema específico
  - Diferença entre aplicativo desktop, web e mobile
  - Exemplos práticos: sistema de cadastro, aplicativo de controle de estoque, app de delivery
  - O ciclo de desenvolvimento: ideia → requisitos → codificação → teste → entrega → manutenção

- **Visão geral das ferramentas do ciclo de desenvolvimento:**
  - **Editor de código / IDE** — onde o código é escrito e testado (ex: VS Code, IntelliJ)
  - **Repositório de código** — onde o código é armazenado e compartilhado (ex: GitHub, GitLab)
  - **Controle de versão** — sistema que registra o histórico de alterações do código (ex: Git)
  - **Ferramentas de teste** — onde erros são detectados antes da entrega (ex: JUnit, PyTest)
  - **Banco de dados** — onde os dados do sistema ficam armazenados (ex: MySQL, PostgreSQL)

- **Por que usar repositório e controle de versão:**
  - Trabalhar em equipe sem sobrescrever o código do colega
  - Voltar para uma versão anterior quando algo dá errado
  - Registrar quem fez o quê e quando — rastreabilidade profissional
  - Portfólio: repositórios públicos no GitHub como vitrine para o mercado de trabalho

- **Ética e profissionalismo desde o início:**
  - Compromisso com prazos e entregas
  - Sigilo sobre projetos e dados dos clientes
  - Organização como competência técnica — código bagunçado é código com problema

## Dinâmica — "De Onde Veio Este App?"

**Duração:** 15 minutos · **Agrupamento:** grupos de 4 a 5 alunos

**Objetivo:** conectar o conteúdo da SA com a realidade dos alunos, identificando as ferramentas e etapas do ciclo de desenvolvimento em aplicativos que eles já usam no cotidiano — desenvolvendo o raciocínio de que todo software foi criado por pessoas com as mesmas ferramentas que eles aprenderão.

**Materiais:** slide com os aplicativos projetados · quadro branco para registrar as respostas dos grupos.

**Passo a passo:**

1. **Apresentação do aplicativo** *(1 min por rodada)*
O professor projeta o ícone ou tela de um aplicativo conhecido e faz a pergunta: *"O que um desenvolvedor precisou para criar isso?"*

2. **Discussão em grupo** *(2 min)*
Cada grupo tenta identificar: qual linguagem pode ter sido usada, onde o código ficaria armazenado, como os dados são guardados e o que poderia dar errado sem testes.

3. **Compartilhamento e conexão** *(2 min)*
Os grupos compartilham suas respostas. O professor conecta cada resposta com as ferramentas apresentadas na aula: IDE, repositório, banco de dados, testes.

**Aplicativos sugeridos para a dinâmica:**

| Aplicativo | Pergunta provocadora | Conexão com o conteúdo |
|------------|---------------------|------------------------|
| iFood / Rappi | *"Quando você faz um pedido, onde esses dados vão parar?"* | Banco de dados · ciclo de desenvolvimento · backend |
| WhatsApp | *"Como duas pessoas em países diferentes trocam mensagem em tempo real?"* | Repositório de código · times de desenvolvimento · controle de versão |
| Instagram | *"Quando você posta uma foto, quantos sistemas diferentes precisam funcionar ao mesmo tempo?"* | Integração de sistemas · IDE · testes |
| Sistema de notas da escola | *"Quem criou o sistema onde o professor lança sua nota?"* | Técnico em desenvolvimento · ciclo completo · IDE e banco de dados |
| Aplicativo do banco | *"Por que o app do banco precisa ser seguro? O que acontece se o código tiver um erro?"* | Ética profissional · qualidade · sigilo de dados |

> **Nota:** o objetivo não é que os alunos respondam certo — é que eles comecem a pensar como desenvolvedores. Toda resposta é válida como ponto de partida. O professor usa as respostas para introduzir naturalmente os conceitos de ferramentas, ciclo de desenvolvimento e responsabilidade profissional.
