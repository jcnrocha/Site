Categorias de Ferramentas no Ciclo de Desenvolvimento

🛠️
Ferramentas de Edição
"Onde o desenvolvedor escreve, executa e depura o código-fonte do projeto"
  • IDE: editor inteligente com debug, autocomplete e extensões
  • Compilador / Interpretador: converte código em execução
  • Linter: analisa e aponta erros de sintaxe em tempo real
  • Formatador: padroniza estilo e indentação automaticamente
  • Ex: VS Code, IntelliJ IDEA, Eclipse, PyCharm
🔀
Controle de Versão
"Registra todo o histórico de alterações do código, permitindo revisitar e reverter qualquer momento"
  • Git: sistema de controle de versão distribuído (local)
  • Commit: snapshot das alterações com mensagem descritiva
  • Branch: linha paralela de desenvolvimento isolada
  • Merge: une alterações de branches diferentes
  • Ex: git init · git commit · git branch · git merge
☁️
Colaboração e Remoto
"Hospeda o repositório na nuvem para acesso da equipe e integração com ferramentas de CI/CD"
  • GitHub: plataforma mais usada para repositórios remotos
  • GitLab: alternativa com CI/CD integrado
  • Bitbucket: popular em ambientes corporativos
  • Pull Request / Merge Request: proposta de integração de código
  • Ex: git push · git pull · git clone · fork · PR

5 Ferramentas Essenciais do Desenvolvedor

🛠️
IDE
Editor inteligente onde o código é escrito, executado e depurado com suporte a extensões
🔀
Git
Sistema de controle de versão que registra o histórico de alterações do projeto localmente
☁️
GitHub
Plataforma remota para hospedar repositórios Git e colaborar com outros desenvolvedores
⌨️
Terminal / CLI
Interface de linha de comando para executar scripts, gerenciar arquivos e rodar comandos Git
📦
Gerenciador de Pacotes
Instala e gerencia bibliotecas externas do projeto — npm, pip, Maven, Composer

Com Controle de Versão (Git) × Sem Controle de Versão

🔀
Com Controle de Versão (Git)
  • Histórico completo de todas as alterações com data, autor e mensagem
  • Reverter para qualquer versão anterior com um único comando
  • Trabalho em equipe simultâneo via branches sem conflitos
  • Código salvo em repositório remoto — sem risco de perda local
  • Rastreabilidade: saber exatamente quem alterou o quê e quando
  • Integração contínua: automatizar testes e deploys a cada commit
Use sempre: essencial em qualquer projeto — individual ou em equipe, acadêmico ou profissional
📁
Sem Controle de Versão
  • Sem histórico — impossível saber o que mudou entre versões
  • Sem reverter — um erro grave pode destruir horas de trabalho
  • Trabalho em equipe por e-mail ou pen drive — propenso a conflitos
  • Código salvo apenas localmente — qualquer falha de hardware = perda total
  • Sem rastreabilidade — ninguém sabe quem fez o quê no projeto
  • Sem automação — testes e publicações são sempre manuais
Evite: mesmo projetos solo se beneficiam do Git — basta git init para começar

🎮 Dinâmica Git na Prática — Qual Comando Usar?
🔀 Git na Prática — Qual Comando Usar?
⏱️ Duração: 15 minutos
👥 Agrupamento: grupos de 4 a 5 alunos
🎯 Objetivo: associar os comandos Git às situações reais do dia a dia do desenvolvedor, fixando o vocabulário e o fluxo de trabalho
1
Distribuição dos cenários (1 min) — Cada grupo recebe 8 cartões (ou vê os slides projetados) com situações reais do desenvolvedor. O desafio: "Qual comando Git você usaria nesta situação?"
2
Discussão em grupo (7 min) — O grupo discute e registra o comando correto para cada cenário; grupos que terminam antes podem criar um cenário extra para desafiar os outros
3
Gabarito e reforço (7 min) — Professor projeta cada cenário, pergunta a resposta dos grupos e explica o uso correto do comando, destacando variações e boas práticas de mensagem de commit
Situação do desenvolvedorComando GitO que o comando faz
Você acabou de criar uma pasta vazia para seu novo projeto e quer começar a rastrear as alterações com Git git init Inicializa um repositório Git local na pasta atual
Você quer baixar um repositório do GitHub para a sua máquina e já ter todo o histórico do projeto git clone <url> Copia o repositório remoto completo para a máquina local
Você modificou 3 arquivos e quer prepará-los para serem incluídos no próximo commit git add . Adiciona todas as alterações à área de stage (índice)
Você terminou uma funcionalidade e quer salvar um snapshot com a mensagem "feat: cadastro de usuário" git commit -m Registra as alterações staged no histórico com uma mensagem
Você quer enviar seus commits locais para o repositório no GitHub para que sua equipe possa ver git push Envia os commits locais para o repositório remoto
Seu colega fez alterações no GitHub e você quer atualizar sua máquina com o que ele enviou git pull Baixa e integra as alterações do repositório remoto
Você quer ver quais arquivos foram modificados mas ainda não foram adicionados ao stage git status Mostra o estado atual da árvore de trabalho e do índice
Você quer ver todos os commits já feitos no projeto com data, autor e mensagem git log Exibe o histórico completo de commits do repositório
💡 Reforço pedagógico: o Git não é apenas uma ferramenta técnica — é um hábito profissional. "Todo commit é uma decisão consciente: o que mudei, por quê mudei e quando mudei." Estimular boas mensagens de commit desde o primeiro dia: use verbos no imperativo (feat, fix, refactor, docs) e seja descritivo o suficiente para que um colega entenda sem precisar abrir o código.
