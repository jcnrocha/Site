Conceito central da aula

🎯 O Que É o Modelo Lógico?

O modelo lógico traduz o DER em estrutura de tabelas — nomes, colunas, PKs e FKs — mas ainda sem código SQL e sem tipos de dados específicos de um SGBD.

Notação textual Como ele é escrito
TABELA (coluna PK, coluna, coluna FK → OUTRA) — formato padrão para representar o esquema sem desenhar.
Posição na modelagem Entre o DER e o SQL
O modelo lógico é a ponte: pega o DER abstrato e prepara o terreno para a fase Física (SA 19).

Lembrete: cada decisão errada aqui se propaga direto para o banco físico — é por isso que essa conversão exige atenção.

Antes das regras de conversão

🔑 Três Tipos de Chave

TipoSignificadoExemplo
CandidataQualquer atributo que poderia identificar a entidadecpf ou matricula do membro
Primária (PK)A chave candidata escolhida para identificar de fatoid_membro
Estrangeira (FK)Referencia a PK de outra tabela — cria o relacionamentoid_membro em EMPRESTIMO

Destaque: toda FK aponta para a PK de outra tabela — é assim que o banco "sabe" como as informações se conectam.

Passo a passo · convertendo entidades

📋 Regras — Entidades e Atributos

Toda conversão de entidade segue a mesma sequência de regras, independente do sistema.

01
Cada entidade vira uma tabela — LIVRO, AUTOR, MEMBRO e CATEGORIA viram 4 tabelas.
02
Cada atributo simples vira uma coluna — mesmo nome, mesma posição na tabela.
03
O atributo identificador vira a PK — a chave sublinhada no DER se torna a chave primária.
04
Atributo composto: cada subatributo vira uma coluna separada na mesma tabela.
05
Atributo multivalorado: vira uma tabela nova, com FK apontando para a entidade principal.
Regra 5 em detalhe · um caso comum

💻 Multivalorado Vira Tabela Própria

Um atributo multivalorado não pode virar uma coluna comum — uma coluna só guarda um valor por linha.

✗ Não funciona
CLIENTE (cpf PK, nome,
  telefones)
-- e se o cliente tiver 3 telefones?
✔ Solução correta
CLIENTE (cpf PK, nome)
TELEFONE_CLIENTE (
  cpf_cliente FK → CLIENTE,
  telefone)

Conclusão: agora cada telefone é uma linha própria — o cliente pode ter quantos telefones precisar.

A segunda parte das regras

📊 Regras — Relacionamentos por Cardinalidade

CardinalidadeO que aconteceNotação
1:1FK vai para a tabela de participação parcialB (pk, fk→A)
1:NFK vai sempre para o lado N — o "muitos"B (pk, fk→A)
N:NCria-se tabela associativa com as duas FKsAB (fk→A, fk→B)

Destaque: em 1:N, a pergunta que resolve tudo é "qual lado é o N?" — é nele que a FK entra.

Aplicando a regra do 1:N

🎯 MEMBRO Realiza EMPRESTIMO — Conversão

MEMBRO (1) realiza EMPRESTIMO (N) — pela regra, a FK vai para o lado N, ou seja, para EMPRESTIMO.

DER conceitual
MEMBRO (1) — realiza → EMPRESTIMO (N), com data_saida e data_devolucao como atributos do relacionamento.
Esquema lógico
EMPRESTIMO (id_emprestimo PK, id_membro FK→MEMBRO, id_livro FK→LIVRO, data_saida, data_devolucao)

Atenção: os atributos do relacionamento (datas) vão para a tabela que recebeu as FKs — nunca ficam soltos.

Aplicando a regra do N:N

🔀 LIVRO É Escrito por AUTOR — Conversão

LIVRO (N) ↔ AUTOR (N) — nenhuma FK simples resolve isso. É preciso criar uma tabela nova.

LIVRO_AUTOR (
  id_livro FK → LIVRO,
  id_autor FK → AUTOR
)
Por que funciona
Cada linha de LIVRO_AUTOR representa uma combinação válida — um livro pode aparecer em várias linhas, com autores diferentes.
Chave composta
As duas FKs juntas formam a chave primária da tabela associativa — a combinação (id_livro, id_autor) é única.
Aplicando ao Projeto Integrador

🎯 Esquema Lógico Completo da Biblioteca

Juntando todas as regras, este é o esquema lógico completo do sistema da Biblioteca — sua entrega da SA 18.

CATEGORIA (id_categoria PK, nome, descricao)
LIVRO (id_livro PK, titulo, isbn, ano_publicacao, id_categoria FK→CATEGORIA)
AUTOR (id_autor PK, nome, nacionalidade, data_nascimento)
LIVRO_AUTOR (id_livro FK→LIVRO, id_autor FK→AUTOR)
MEMBRO (id_membro PK, nome, matricula, email, telefone)
EMPRESTIMO (id_emprestimo PK, id_membro FK→MEMBRO, id_livro FK→LIVRO, data_saida, data_devolucao)

Destaque: 6 tabelas no total — 4 entidades originais + 1 tabela associativa (LIVRO_AUTOR) para o N:N.
