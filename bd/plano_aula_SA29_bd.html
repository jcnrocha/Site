Visão geral dos tipos de JOIN
⬛ INNER JOIN esta aula
Retorna apenas os registros que têm correspondência nas duas tabelas. Registros sem par ficam de fora do resultado.
◧ LEFT JOIN SA 30
Retorna todos os registros da tabela da esquerda, mesmo que não tenham correspondência na tabela da direita. O lado direito aparece como NULL quando não há par.
◨ RIGHT JOIN SA 30
Retorna todos os registros da tabela da direita, mesmo que não tenham correspondência na tabela da esquerda. O lado esquerdo aparece como NULL quando não há par.
Sintaxe do INNER JOIN — sem e com alias
Sem alias — nomes completos
SELECT cliente.nome, locacao.data_locacao
FROM cliente
INNER JOIN locacao
  ON cliente.cpf = locacao.cpf_cliente;
Com alias — código mais limpo
SELECT c.nome, l.data_locacao
FROM cliente c
INNER JOIN locacao l
  ON c.cpf = l.cpf_cliente;
Encadeando múltiplos JOINs
-- três tabelas: cliente → locacao → filme
SELECT c.nome, f.titulo, l.data_locacao
FROM cliente c
INNER JOIN locacao l ON c.cpf = l.cpf_cliente
INNER JOIN filme f ON l.id_filme = f.id_filme
ORDER BY l.data_locacao DESC;
INNER JOIN + GROUP BY + COUNT
-- quantas locações cada cliente realizou?
SELECT c.nome, COUNT(*) AS total_locacoes
FROM cliente c
INNER JOIN locacao l ON c.cpf = l.cpf_cliente
GROUP BY c.nome
ORDER BY total_locacoes DESC;

1
Mostre o nome do cliente e a data de cada locação realizada
CLIENTE ⟶ LOCACAO
Gabarito esperado
SELECT c.nome, l.data_locacao
FROM cliente c
INNER JOIN locacao l ON c.cpf = l.cpf_cliente;
2
Mostre o nome do cliente e o título do filme de cada locação
CLIENTE ⟶ LOCACAO ⟶ FILME
Gabarito esperado
SELECT c.nome, f.titulo
FROM cliente c
INNER JOIN locacao l ON c.cpf = l.cpf_cliente
INNER JOIN filme f ON l.id_filme = f.id_filme;
3
Mostre o título do filme e o nome dos atores que participaram de cada filme
FILME ⟶ ELENCO ⟶ ATOR (via tabela associativa)
Gabarito esperado
SELECT f.titulo, a.nome
FROM filme f
INNER JOIN elenco e ON f.id_filme = e.id_filme
INNER JOIN ator a ON e.id_ator = a.id_ator;
4
Mostre quantas locações cada cliente realizou, exibindo o nome do cliente e o total — ordenado do maior para o menor
CLIENTE ⟶ LOCACAO + GROUP BY + COUNT
Gabarito esperado
SELECT c.nome, COUNT(*) AS total
FROM cliente c
INNER JOIN locacao l ON c.cpf = l.cpf_cliente
GROUP BY c.nome
ORDER BY total DESC;
💡 Ponto pedagógico — rodada 4: reforçar que COUNT, GROUP BY e ORDER BY continuam funcionando exatamente como na SA 28 — o JOIN apenas expande o alcance do SELECT, conectando o que a FK definiu no modelo lógico. Ressaltar o uso do alias de tabela como boa prática que torna o código mais legível.
⚠️ Atenção — INNER JOIN exclui registros sem par: se um cliente nunca fez locação, ele não aparece no resultado do INNER JOIN. Quando isso é um problema, usa-se o LEFT JOIN — conteúdo da próxima aula (SA 30).
