📚 Conteúdo da Aula INSERT INTO · Tipos · Restrições
Com lista de colunas vs. sem lista de colunas
✅ Com lista de colunas — recomendado
INSERT INTO produto (nome, preco)
VALUES ('Teclado', 150.00);
Colunas explícitas — seguro, legível e independente da ordem das colunas na tabela. Colunas com DEFAULT ou AUTO_INCREMENT podem ser omitidas.
⚠️ Sem lista de colunas — arriscado
INSERT INTO produto
VALUES (NULL, 'Teclado', 150.00, 10, TRUE);
Todos os valores obrigatórios na ordem exata das colunas. Qualquer alteração no CREATE TABLE quebra silenciosamente o INSERT.
Inserção de múltiplos registros
-- inserir vários registros de uma só vez
INSERT INTO produto (nome, preco, estoque)
VALUES
    ('Teclado', 150.00, 20),
    ('Mouse', 80.00, 35),
    ('Monitor', 1200.00, 8);
Formato dos valores por tipo de dado
Tipo
Formato correto
Observação
Numérico 42  ·  150.00  ·  -5 Sem aspas — ponto como separador decimal
Texto 'João Silva'  ·  'SP' Entre aspas simples obrigatoriamente
Data '2026-04-15'  ·  '2026-04-15 10:30:00' Aspas simples · formato AAAA-MM-DD
Booleano TRUE  ·  FALSE Sem aspas — MySQL armazena como 1 ou 0
NULL NULL Sem aspas — indica ausência explícita de valor
Como as restrições do CREATE TABLE impactam o INSERT INTO
NOT NULL
Impede inserção sem o valor da coluna. Se omitida e sem DEFAULT, o banco retorna erro.
ERROR: Field 'nome' doesn't have a default value
UNIQUE
Rejeita qualquer tentativa de inserir um valor já existente nessa coluna.
ERROR: Duplicate entry 'joao@email.com' for key 'email'
PRIMARY KEY
Impede inserção de chave primária repetida ou nula. Cada linha deve ter um ID único.
ERROR: Duplicate entry '1' for key 'PRIMARY'
DEFAULT valor
Quando a coluna é omitida no INSERT, o banco assume automaticamente o valor padrão — sem necessidade de informá-lo.
estoque INT DEFAULT 0 → omitir = banco insere 0
AUTO_INCREMENT
O valor é gerado automaticamente — não deve ser informado no INSERT. Se informado como NULL, o banco gera o próximo valor da sequência.
id INT AUTO_INCREMENT → não incluir na lista de colunas
Erros comuns no INSERT INTO
  • 🚫
    Tipo incompatível — inserir texto em coluna numérica: preco = '1200,00' em vez de 1200.00
  • 🚫
    Coluna obrigatória omitida — esquecer coluna com NOT NULL e sem DEFAULT
  • 🚫
    Valor duplicado — inserir email ou CPF já existente em coluna UNIQUE ou PRIMARY KEY
  • 🚫
    Ordem incorreta — sintaxe sem lista de colunas com valores fora da sequência da tabela
  • 🚫
    Aspas faltando — VALUES (João, 80.00) em vez de VALUES ('João', 80.00)

Tabela de contexto — exibir antes das rodadas
-- esta é a tabela usada em todos os INSERTs da dinâmica
CREATE TABLE produto (
    id_produto  INT          PRIMARY KEY AUTO_INCREMENT,
    nome        VARCHAR(100) NOT NULL,
    preco       DECIMAL(8,2) NOT NULL,
    estoque     INT          DEFAULT 0,
    ativo       BOOLEAN      DEFAULT TRUE
);
1
INSERT com colunas explícitas — omite id_produto e defaults
✅ Correto
INSERT INTO produto (nome, preco) VALUES ('Teclado', 150.00);
Tudo certo
Colunas com AUTO_INCREMENT e DEFAULT podem ser omitidas quando se usa lista explícita de colunas. O banco assume os padrões automaticamente.
2
INSERT sem lista de colunas — número de valores incorreto
❌ Errado
INSERT INTO produto VALUES ('Mouse', 80.00, 10, TRUE);
Problema identificado
Sintaxe sem lista de colunas exige todos os valores na ordem exata das colunas da tabela. A tabela produto tem 5 colunas, mas apenas 4 valores foram fornecidos — falta o id_produto (deve ser NULL para acionar o AUTO_INCREMENT).
3
INSERT com preço como texto com vírgula
❌ Errado
INSERT INTO produto (nome, preco, estoque)
VALUES ('Monitor', '1200,00', 5);
Problema identificado
Tipo incompatível — '1200,00' é uma string com vírgula, não um número decimal. A coluna preco é DECIMAL(8,2). O correto é 1200.00 sem aspas e com ponto como separador decimal.
4
INSERT de múltiplos registros — sintaxe correta
✅ Correto
INSERT INTO produto (nome, preco) VALUES ('Headset', 250.00),        ('Webcam', 199.90),        ('Hub USB', 89.50);
Tudo certo
Inserção de múltiplos registros com lista de colunas explícita e valores separados por vírgula. Sintaxe válida e eficiente — um único INSERT para três linhas.
💡 Ponto pedagógico: as rodadas 1 e 4 são intencionalmente corretas. O objetivo é equilibrar identificação de erro com reconhecimento de código correto — evitando que os alunos votem ❌ por precaução sem analisar o código com atenção.
