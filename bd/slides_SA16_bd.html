Conceito central da aula

🎯 O Que É uma Entidade?

Uma entidade representa um objeto do mundo real com existência própria sobre o qual queremos guardar informações — uma "coisa" do domínio do sistema.

Entidade × Instância São coisas diferentes
CLIENTE é a entidade (o conceito); "João Silva" é uma instância dela (um caso real).
Forte × Fraca Dois tipos de entidade
Entidade forte existe por si só; entidade fraca depende de outra para existir (ex: dependente de funcionário).

No projeto da Biblioteca: LIVRO, AUTOR, MEMBRO e CATEGORIA são candidatas a entidades fortes.

Depois de identificar a entidade

🎯 O Que É um Atributo?

Um atributo é uma característica ou propriedade que descreve a entidade — cada instância da entidade tem seu próprio valor para esse atributo.

Exemplo
A entidade LIVRO pode ter os atributos titulo, isbn e ano_publicacao.
Atributo identificador
Todo atributo identificador (chave) precisa ser único para cada instância — é como localizamos um registro específico.

Dica: pergunte sempre "isso varia de instância para instância?" — se sim, é atributo; se for sempre igual para todas, talvez não seja.

Os quatro tipos · visão geral

📊 Simples, Composto, Multivalorado, Derivado

TipoSignificadoExemplo
SimplesValor único e indivisívelcpf · ano · nome
CompostoFormado por subatributosendereco → rua, nº, CEP
MultivaloradoPode ter mais de um valortelefones · emails
DerivadoCalculado a partir de outro atributoidade ← data_nascimento

Destaque: classificar corretamente cada atributo é o que vai garantir um DER fiel à realidade do sistema.

Tipos 1 e 2 de 4

◉ Simples × ◈ Composto

◉ Simples
→ Não pode ser dividido em partes menores
→ Exemplos: cpf, ano, nome
→ É o tipo mais comum em qualquer DER
◈ Composto
→ Formado por vários subatributos juntos
→ Exemplo: endereco → rua, número, CEP
→ No DER, ramifica em sub-elipses

No projeto da Biblioteca: titulo é simples; um futuro atributo endereco_membro seria composto.

Tipos 3 e 4 de 4

◉◉ Multivalorado × ◌ Derivado

◉◉ Multivalorado
→ Pode ter vários valores na mesma instância
→ Exemplo: um membro com vários telefones
→ No DER, elipse com borda dupla
◌ Derivado
→ Calculado a partir de outro atributo já existente
→ Exemplo: idade calculada de data_nascimento
→ No DER, elipse com borda tracejada

Atenção: nunca armazene um atributo derivado diretamente — calcule-o sempre a partir da fonte original.

Como fazer na prática

📋 Passo a Passo: Identificando Entidades e Atributos

Use essa sequência sempre que precisar identificar entidades a partir de uma descrição de sistema.

01
Leia a descrição e sublinhe os substantivos importantes — eles são candidatos a entidade.
02
Filtre o que é realmente entidade — descarte o que é só uma característica de outra coisa (isso é atributo, não entidade).
03
Liste os atributos de cada entidade e classifique: simples, composto, multivalorado ou derivado.
04
Escolha o atributo identificador — o que torna cada instância única dentro daquela entidade.
