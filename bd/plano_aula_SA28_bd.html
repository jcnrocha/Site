As 5 funções de agregação
COUNTContagem
Conta o número de registros. COUNT(*) inclui nulos; COUNT(col) ignora nulos.
COUNT(*) AS total
COUNT(email) AS com_email
SUMSoma
Soma os valores numéricos de uma coluna. Ignora valores nulos automaticamente.
SUM(valor) AS total_vendas
AVGMédia
Calcula a média aritmética dos valores de uma coluna numérica. Ignora nulos.
AVG(ano) AS media_ano
MINMínimo
Retorna o menor valor da coluna. Funciona com números, datas e textos (ordem alfabética).
MIN(ano) AS mais_antigo
MAXMáximo
Retorna o maior valor da coluna. Funciona com números, datas e textos (ordem alfabética).
MAX(ano) AS mais_recente
GROUP BY — agrupando registros
-- regra: toda coluna no SELECT que não é função deve estar no GROUP BY
SELECT genero, COUNT(*) AS total_filmes
FROM filme
GROUP BY genero
ORDER BY total_filmes DESC;
WHERE vs. HAVING — a distinção fundamental
🔍 WHERE — filtra linhas
Executa ANTES da agregação
Elimina linhas individuais da tabela antes de qualquer agrupamento ou contagem. Não pode usar funções de agregação.
SELECT genero, COUNT(*)
FROM filme
WHERE ano > 2010 -- filtra linhas
GROUP BY genero;
📊 HAVING — filtra grupos
Executa DEPOIS da agregação
Elimina grupos inteiros após o agrupamento. Pode usar funções de agregação — é exatamente por isso que existe.
SELECT genero, COUNT(*) AS total
FROM filme
GROUP BY genero
HAVING COUNT(*) > 5; -- filtra grupos
Ordem de execução completa
① FROM
carrega a tabela
→
② WHERE
filtra linhas
→
③ GROUP BY
agrupa registros
→
④ HAVING
filtra grupos
→
⑤ SELECT
projeta colunas
→
⑥ ORDER BY
ordena resultado

Gabarito esperado
SELECT genero, COUNT(*) AS total
FROM filme
GROUP BY genero
HAVING COUNT(*) > 3;
💡 Ponto pedagógico — rodada 4: reforçar por que não é possível usar WHERE COUNT(*) > 3 — o COUNT ainda não existe no momento em que o WHERE é executado (passo ② na ordem de execução). Usar o fluxo de execução para explicar visualmente.
