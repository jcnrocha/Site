O que é uma subquery?

Uma subquery (ou subconsulta) é um SELECT aninhado dentro de outro SELECT, sempre delimitado por parênteses. A consulta interna é avaliada primeiro e seu resultado é usado pela consulta externa.

WHERE com IN lista de valores
A subquery retorna múltiplos valores. O IN verifica se o campo externo está entre eles. Ideal quando a lista é dinâmica — calculada pelo banco, não digitada.
WHERE escalar único valor
A subquery retorna um único valor (ex.: média, máximo). Usa operadores = > < diretamente. O banco avalia e substitui o parêntese pelo número.
FROM (tabela derivada) pré-processamento
A subquery no FROM age como uma tabela temporária. Deve receber um alias obrigatório. Permite filtrar, agregar e depois consultar o resultado como se fosse uma tabela real.
Correlacionada linha a linha
A subquery referencia a tabela externa — é executada uma vez para cada linha da consulta de fora. Mais poderosa, mas também mais pesada. Usada com EXISTS e em comparações que dependem do contexto da linha atual.
Subquery no WHERE com IN
-- clientes que alugaram pelo menos um filme do gênero 'Ação'
SELECT nome, email
FROM cliente
WHERE cpf IN (
    SELECT cpf_cliente
    FROM locacao l
    JOIN filme f ON l.id_filme = f.id_filme
    WHERE f.genero = 'Ação'
);
Subquery escalar no WHERE
-- filmes lançados depois do ano médio do catálogo
SELECT titulo, ano
FROM filme
WHERE ano > (SELECT AVG(ano) FROM filme)
ORDER BY ano DESC;
Subquery no FROM — tabela derivada
-- top clientes: nome e total de locações, filtrando só quem fez mais de 2
SELECT resumo.nome, resumo.total
FROM (
    SELECT c.nome, COUNT(*) AS total
    FROM cliente c
    INNER JOIN locacao l ON c.cpf = l.cpf_cliente
    GROUP BY c.nome
) resumo
WHERE resumo.total > 2
ORDER BY resumo.total DESC;
💡 Tabela derivada precisa de alias: o resultado do subquery no FROM é tratado pelo banco como uma tabela temporária — sem alias ele não sabe como referenciá-la. resumo no exemplo acima é obrigatório.
Subquery vs JOIN — quando usar cada um?
Situação Prefira Motivo
Preciso de colunas das duas tabelas no resultado JOIN Subquery no WHERE não expõe colunas da tabela interna
Filtro baseado em lista calculada dinamicamente Subquery IN Mais legível que JOIN + GROUP BY quando só preciso filtrar
Comparar com agregação (média, máximo…) Subquery escalar JOIN não resolve diretamente comparações com agrega­ções
Filtrar depois de uma agregação complexa Subquery no FROM Pré-processa o agrupamento antes de aplicar o filtro

CLIENTE ⟶ LOCACAO + FILME NOT IN (subquery)
Gabarito esperado
-- NOT IN: exclui quem já aparece na lista de quem alugou Comédia
SELECT nome
FROM cliente
WHERE cpf NOT IN (

);
3
Mostre os filmes lançados depois do ano médio de lançamento de todos os filmes do catálogo — exiba título e ano
FILME WHERE > escalar (AVG)
Gabarito esperado
-- parte interna retorna um único número: a média dos anos
SELECT titulo, ano
FROM filme
WHERE ano > (SELECT AVG(ano) FROM filme)
ORDER BY ano DESC;
4
Liste o nome e o total de locações de cada cliente, mas exiba apenas os que fizeram mais de 2 locações
CLIENTE + LOCACAO Subquery no FROM
Gabarito esperado
-- tabela derivada agrupa e conta; SELECT externo aplica o filtro de total
SELECT s.nome, s.total
FROM (
    SELECT c.nome, COUNT(*) AS total
    FROM cliente c
    INNER JOIN locacao l ON c.cpf = l.cpf_cliente
    GROUP BY c.nome
) s
WHERE s.total > 2
ORDER BY s.total DESC;
💡 Ponto pedagógico — rodada 4: reforçar por que não podemos usar WHERE COUNT(*) > 2 diretamente — funções de agregação não são permitidas no WHERE (apenas no HAVING, mas HAVING filtra grupos, não linhas). A subquery no FROM resolve isso processando o agrupamento primeiro.
⚠️ Cuidado com NOT IN e NULL: se a subquery interna retornar algum valor NULL, o NOT IN pode retornar zero linhas — comportamento silencioso e difícil de depurar. Boa prática: adicionar WHERE coluna IS NOT NULL na subquery interna quando houver risco de NULL.
🔁 Subconsulta correlacionada: apresentar brevemente como curiosidade — o aluno perceberá o padrão em situações futuras. A principal distinção é que a subquery interna referencia uma coluna da tabela externa, tornando-a dependente de cada linha processada. Não exigir escrita desta forma na avaliação desta SA.
