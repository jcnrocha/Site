Parte 1 — Questões Objetivas 5 questões · 1,0 pt cada · total 5,0 pts
01
Considere a consulta abaixo. O que ela retorna?
SELECT c.nome, l.data_locacao
FROM cliente c
LEFT JOIN locacao l ON c.cpf = l.cpf_cliente
WHERE l.id_locacao IS NULL;
1,0 pt
  • AO nome de todos os clientes e a data de sua última locação
  • BO nome dos clientes que realizaram pelo menos uma locação
  • CO nome dos clientes que nunca realizaram nenhuma locação
  • DUm erro de execução, pois não é possível filtrar por IS NULL após LEFT JOIN
02
Qual é o risco de usar NOT IN com uma subquery que pode retornar valores NULL?
1,0 pt
  • AA consulta retorna um erro de execução imediato
  • BOs valores NULL são ignorados automaticamente e a consulta funciona normalmente
  • CA consulta executa, mas retorna zero linhas se a subquery contiver algum NULL
  • DOs valores NULL são tratados como zero (0) na comparação com NOT IN
03
Qual das alternativas lista corretamente apenas os gêneros de filmes que possuem mais de 3 títulos cadastrados?
1,0 pt
  • ASELECT genero, COUNT(*) FROM filme WHERE COUNT(*) > 3 GROUP BY genero
  • BSELECT genero, COUNT(*) FROM filme GROUP BY genero HAVING COUNT(*) > 3
  • CSELECT genero, COUNT(*) FROM filme HAVING COUNT(*) > 3
  • DSELECT genero, COUNT(*) FROM filme GROUP BY genero WHERE COUNT(*) > 3
04
Qual afirmação sobre Views em SQL está correta?
1,0 pt
  • AUma VIEW armazena os dados fisicamente no banco — consultas à view são mais rápidas que a query original
  • BUma VIEW é uma query nomeada — não armazena dados e é recalculada a cada SELECT sobre ela
  • CNão é possível usar JOIN dentro de uma VIEW
  • DO comando DROP VIEW elimina a view e também todas as tabelas que ela referencia
05

Q06 LEFT JOIN + COUNT — clientes com total de locações incluindo zero 1,0 pt
Critérios: LEFT JOIN correto (0,3) · COUNT(l.id_locacao) — não COUNT(*) para contar NULL como 0 (0,3) · GROUP BY c.cpf e c.nome (0,2) · ORDER BY correto (0,2)
-- total de locações por cliente, incluindo quem nunca locou
SELECT c.nome,
         COUNT(l.id_locacao) AS total_locacoes
FROM cliente c
LEFT JOIN locacao l ON c.cpf = l.cpf_cliente
GROUP BY c.cpf, c.nome
ORDER BY total_locacoes DESC;
💡 Ponto de atenção: COUNT(*) contaria o NULL do LEFT JOIN como 1 — o aluno deve usar COUNT(l.id_locacao) para que clientes sem locação apareçam com 0. Se o aluno usou COUNT(*) mas o restante está correto, desconto de 0,2 pt.
Q07 Subquery scalar — filmes com valor_diaria acima da média 1,0 pt
Critérios: subquery com AVG correta (0,4) · operador > correto (0,2) · SELECT com titulo e valor_diaria (0,2) · FROM filme correto (0,2)
-- filmes com diária acima da média do catálogo
SELECT titulo, valor_diaria
FROM filme
WHERE valor_diaria > (
    SELECT AVG(valor_diaria)
    FROM filme
);
Q08 CREATE VIEW + consulta sobre a view com filtro de gênero 1,0 pt
Critérios: CREATE OR REPLACE VIEW com nome correto (0,2) · JOINs corretos para juntar as 3 tabelas (0,3) · colunas corretas na view (0,2) · SELECT sobre a view com WHERE genero (0,3)
-- view reutilizável para consultas de locação detalhada
CREATE OR REPLACE VIEW vw_locacoes_detalhadas AS
SELECT c.nome,
           f.titulo, f.genero,
           l.data_locacao
FROM locacao l
INNER JOIN cliente c ON l.cpf_cliente = c.cpf
INNER JOIN filme f ON l.id_filme = f.id_filme;

-- consultando a view com filtro de gênero
SELECT * FROM vw_locacoes_detalhadas
WHERE genero = 'Ação';
Q09 UPDATE com transação completa — BEGIN · UPDATE · SELECT · COMMIT 1,0 pt
Critérios: BEGIN presente (0,2) · UPDATE com SET e WHERE corretos (0,3) · SELECT de verificação antes do COMMIT (0,2) · COMMIT presente (0,2) · menção a ROLLBACK como alternativa (0,1 — bônus, não obrigatório)
BEGIN;

-- verificar o registro antes de alterar
SELECT * FROM cliente
WHERE cpf = '111.222.333-44';

UPDATE cliente
SET cidade_cliente = 'Curitiba'
WHERE cpf = '111.222.333-44'; -- sem WHERE alteraria todos

-- confirmar resultado antes de persistir
SELECT * FROM cliente
WHERE cpf = '111.222.333-44';

COMMIT; -- ou ROLLBACK se algo estiver incorreto
Q10 Reescrita de comentários — remover óbvios e adicionar os que explicam o porquê 1,0 pt
Critérios: remover todos os 5 comentários ruins (0,4 — 0,08 por comentário removido) · adicionar pelo menos um comentário útil que explique o propósito ou a regra de negócio (0,4) · código correto e sem alterações na lógica (0,2)
-- catálogo de filmes de Ação ordenados do mais caro ao mais barato
