Parte 1 — Questões Objetivas 5 questões · 1,0 pt cada · total 5,0 pts
01
Qual é a diferença entre LEFT JOIN e RIGHT JOIN em SQL?
1,0 pt
  • ALEFT JOIN mantém todas as linhas da tabela da esquerda; RIGHT JOIN mantém todas as da direita
  • BLEFT JOIN e RIGHT JOIN produzem exatamente o mesmo resultado em qualquer cenário
  • CLEFT JOIN só funciona com duas tabelas; RIGHT JOIN aceita qualquer número de tabelas
  • DLEFT JOIN é mais lento que RIGHT JOIN para tabelas com muitos registros
02
Qual consulta retorna os nomes dos clientes que locaram pelo menos um filme do gênero 'Romance' usando subquery com WHERE IN?
1,0 pt
  • ASELECT nome FROM cliente WHERE genero = 'Romance'
  • BSELECT nome FROM cliente WHERE cpf IN (SELECT cpf_cliente FROM locacao l INNER JOIN filme f ON l.id_filme = f.id_filme WHERE f.genero = 'Romance')
  • CSELECT nome FROM cliente INNER JOIN locacao ON cpf = cpf_cliente WHERE genero = 'Romance'
  • DSELECT nome FROM cliente WHERE cpf IN (SELECT * FROM locacao WHERE genero = 'Romance')
03
Qual é a vantagem de usar CREATE OR REPLACE VIEW em vez de CREATE VIEW?
1,0 pt
  • ACREATE OR REPLACE VIEW persiste os dados no disco, tornando consultas mais rápidas
  • BCREATE OR REPLACE VIEW permite usar JOINs dentro da view; CREATE VIEW não permite
  • CCREATE OR REPLACE VIEW cria a view ou a substitui se já existir, sem gerar erro de execução
  • DCREATE OR REPLACE VIEW impede que outros usuários alterem a view após a criação
04
O que acontece ao executar DELETE FROM locacao sem nenhuma cláusula WHERE?
1,0 pt
  • AO banco solicita uma confirmação interativa antes de prosseguir
  • BApenas as locações com id_locacao menor que 100 são removidas
  • CTodos os registros da tabela locacao são removidos permanentemente
  • DO comando retorna erro de sintaxe por não ter a cláusula WHERE
05
Para desativar múltiplas linhas de código SQL com um único comentário — sem apagá-las —, qual é a sintaxe correta?
1,0 pt
  • A# código # envolvendo o bloco
  • B// código // no início e fim do bloco
  • C/* código */ envolvendo o bloco inteiro
  • DDISABLE código ENDDISABLE como palavras reservadas

Q06 INNER JOIN + GROUP BY + HAVING — filmes com 2 ou mais locações 1,0 pt
Critérios: INNER JOIN correto com ON (0,3) · COUNT(*) ou COUNT(l.id_locacao) com alias (0,2) · GROUP BY f.id_filme e f.titulo (0,2) · HAVING COUNT(*) >= 2 (0,2) · ORDER BY DESC (0,1)
-- filmes mais populares na locadora (locados 2 ou mais vezes)
SELECT f.titulo,
         COUNT(*) AS total_locacoes
FROM filme f
INNER JOIN locacao l ON f.id_filme = l.id_filme
GROUP BY f.id_filme, f.titulo
HAVING COUNT(*) >= 2
ORDER BY total_locacoes DESC;
Q07 Subquery WHERE IN — clientes que locaram pelo menos um filme de Ação 1,0 pt
Critérios: WHERE cpf IN (subquery) correto (0,3) · subquery com JOIN entre LOCACAO e FILME (0,3) · filtro f.genero = 'Ação' na subquery (0,2) · SELECT c.nome correto (0,2). DISTINCT é opcional — aceitar sem penalidade.
-- clientes que assistiram pelo menos um filme de Ação
SELECT DISTINCT nome
FROM cliente
WHERE cpf IN (
    SELECT l.cpf_cliente
    FROM locacao l
    INNER JOIN filme f ON l.id_filme = f.id_filme
    WHERE f.genero = 'Ação'
);
Q08 CREATE VIEW vw_filmes_acao + DROP VIEW IF EXISTS 1,0 pt
Critérios: CREATE OR REPLACE VIEW com nome correto (0,2) · SELECT com titulo, ano, valor_diaria (0,2) · WHERE genero = 'Ação' (0,2) · DROP VIEW IF EXISTS com nome correto (0,4)
-- catálogo de filmes de Ação disponíveis para locação
CREATE OR REPLACE VIEW vw_filmes_acao AS
SELECT titulo, ano, valor_diaria
FROM filme
WHERE genero = 'Ação';

-- remoção segura: não gera erro se a view não existir
DROP VIEW IF EXISTS vw_filmes_acao;
Q09 DELETE com transação — BEGIN · SELECT verificação · DELETE com WHERE · COMMIT 1,0 pt
Critérios: BEGIN presente (0,2) · SELECT de verificação antes do DELETE (0,2) · DELETE com WHERE data_locacao < '2024-01-01' (0,3) · COMMIT presente (0,2) · menção a ROLLBACK (0,1 — bônus)
BEGIN;

-- quantos registros serão removidos
SELECT * FROM locacao
WHERE data_locacao < '2024-01-01';

-- sem WHERE removeria todas as locações
DELETE FROM locacao
WHERE data_locacao < '2024-01-01';

-- deve retornar zero linhas se o DELETE funcionou
SELECT * FROM locacao
WHERE data_locacao < '2024-01-01';

COMMIT; -- ou ROLLBACK para desfazer
