🔐 Conteúdo — SQL DCL Data Control Language
O que é DCL e por que importa

DCL — Data Control Language — é a categoria SQL responsável pelo controle de acesso ao banco de dados. Enquanto DDL define estrutura e DML manipula dados, o DCL define quem pode fazer o quê — aplicando o princípio do menor privilégio: cada usuário acessa apenas o que precisa para sua função.

Permissões disponíveis
SELECT INSERT UPDATE DELETE CREATE DROP ALTER
GRANT vs REVOKE
✅ GRANT — Conceder permissão
-- conceder SELECT e INSERT
GRANT SELECT, INSERT
ON loja.produto
TO 'app_user'@'localhost';
Concede as permissões listadas ao usuário especificado no banco/tabela indicado.
🚫 REVOKE — Revogar permissão
-- revogar apenas INSERT
REVOKE INSERT
ON loja.produto
FROM 'app_user'@'localhost';
Remove permissões específicas do usuário — as demais permissões permanecem intactas.
Perfis de acesso em sistemas reais
📊Usuário de Leitura

Usado em dashboards, relatórios e consultas analíticas. Não pode modificar dados.

SELECT INSERT UPDATE DELETE DROP
⚙️Usuário de Aplicação

Usado pelo sistema em produção. Opera dados sem poder alterar estrutura.

SELECT INSERT UPDATE DELETE DROP ALTER
🔑Administrador

Acesso total ao banco. Responsável pela estrutura, backup e segurança.

SELECT INSERT UPDATE DELETE CREATE DROP ALTER
