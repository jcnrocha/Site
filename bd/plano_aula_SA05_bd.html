📚 Conteúdo da Aula SGBD · Funções · Mercado
BD × SGBD — Dois componentes distintos
🗄️
Banco de Dados
BD
Conjunto organizado de dados armazenados em disco. É o repositório — onde as informações ficam gravadas de forma estruturada e persistente. Não faz nada sozinho.
≠
⚙️
SGBD
Sistema de Gerenciamento de Banco de Dados
Software que gerencia o acesso ao banco de dados. É o intermediário entre a aplicação e os dados — processa consultas, controla acesso, garante integridade e segurança.
Funções Principais do SGBD
🏗️
Definição de Dados
DDL
Permite criar, alterar e remover a estrutura do banco — tabelas, colunas, índices e restrições.
✏️
Manipulação de Dados
DML · DQL
Permite inserir, atualizar, excluir e consultar dados. É a função mais usada no dia a dia do desenvolvedor.
🔐
Controle de Acesso
DCL
Define quem pode fazer o quê no banco — concessão e revogação de permissões por usuário ou perfil.
♻️
Recuperação
Backup
Restaura o banco a um estado consistente após falhas, quedas de energia ou erros de transação.
🔄
Controle de Concorrência
Lock
Gerencia múltiplos acessos simultâneos garantindo que operações paralelas não gerem conflito ou inconsistência.
✅
Integridade
Constraints
Garante que os dados respeitam as regras definidas — chaves, tipos, restrições NOT NULL e referências externas.
Tipos de SGBD
Relacional
RDBMS
  • Dados em tabelas com linhas e colunas
  • Relacionamentos por chaves estrangeiras
  • Linguagem SQL padronizada
  • MySQL · PostgreSQL · Oracle · SQL Server
NoSQL
Not Only SQL
  • Modelos flexíveis: documento, chave-valor, coluna, grafo
  • Alta escalabilidade horizontal
  • Sem esquema rígido
  • MongoDB · Redis · Cassandra · Neo4j
NewSQL
Híbrido
  • Escalabilidade do NoSQL com transações do relacional
  • Mantém suporte a SQL completo
  • Para sistemas de alta disponibilidade
  • CockroachDB · Google Spanner

🎮 Dinâmica Quem Sou Eu? — Round SGBD
🔍 Quem Sou Eu? — Identificação por Pistas
⏱️ Duração: 15 minutos
👥 Agrupamento: grupos de 4 a 5 alunos
🏆 Pontuação: 3 pts na 1ª pista · 2 pts na 2ª · 1 pt na 3ª
1
Leitura das pistas (1 min/rodada) — Professor lê as pistas em ordem crescente de especificidade. Grupos tentam identificar o SGBD após cada pista.
2
Resposta em grupo (1 min) — O grupo que identificar primeiro levanta a mão. Pontuação conforme a pista em que acertou.
3
Consolidação (1 min) — Professor confirma a resposta e reforça as características do SGBD identificado.
# Pista 1 3 pts Pista 2 2 pts Pista 3 1 pt Resposta
1 Sou relacional e código aberto Sou muito usado em aplicações web com PHP e WordPress Sou mantido pela Oracle e meu nome tem a ver com o nome de uma filha do meu criador MySQL
2 Sou relacional e conhecido pela minha robustez Sou forte em conformidade com padrões SQL e extensibilidade Meu nome significa "elefante" e meu logo é um elefante azul PostgreSQL
3 Sou um banco de dados que não precisa de servidor Sou embutido direto na aplicação e muito usado em apps mobile Estou presente em todos os iPhones e aparelhos Android sem que o usuário saiba SQLite
4 Sou NoSQL e guardo dados no formato de documentos JSON Sou muito usado em aplicações que precisam de alta escalabilidade Meu nome vem da palavra francesa que significa "canhão" MongoDB
5 Sou NoSQL e especializado em guardar pares de chave e valor Sou extremamente rápido porque guardo os dados em memória Sou muito usado para cache de sessões e filas de mensagens Redis
