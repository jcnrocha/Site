Os dois tipos de comentário SQL
-- Comentário de linha
Ignora tudo da posição -- até o final da linha. Ideal para anotações curtas ao lado do código ou para desativar uma linha temporariamente.
-- retorna clientes ativos apenas
SELECT * FROM cliente
WHERE ativo = 1; -- 0 = inativo
/* */ Comentário de bloco
Ignora tudo entre /* e */, podendo ocupar várias linhas. Usado no cabeçalho de scripts e para desativar blocos inteiros de código.
/*
  Script: carga inicial da locadora
  Autor: Prof. Julio · Data: 28/10/2026
*/
Cabeçalho padrão de script
/*
  Banco de dados: locadora
  Script: consultas de relatório mensal
  Propósito: geração do relatório de locações por cliente — outubro/2026
  Autor: Julio Cesar · Data: 28/10/2026
  Dependências: tabelas cliente, locacao, filme
*/
O que comentar e o que não comentar
✔ Comente o POR QUÊ
  • Regras de negócio não-óbvias no WHERE
  • Constraints com significado específico
  • Joins que exigem lógica de domínio
  • Workarounds e limitações do sistema
  • Cabeçalho com contexto do script
  • Código desativado temporariamente
✘ Não comente o ÓBVIO
  • Repetir em palavras o que o código diz
  • Nomes de tabelas e colunas já claros
  • Cada linha do SELECT individualmente
  • Comentários desatualizados (mentem!)
  • TODO que nunca será resolvido
  • Histórico de versão (use git para isso)
Comparação — comentário ruim vs comentário útil
✘ Comentário ruim — repete o código
-- seleciona nome e email do cliente
SELECT nome, email
FROM cliente
-- filtra por ativo igual a 1
WHERE ativo = 1;
✔ Comentário útil — explica o porquê
-- destinatários do e-mail marketing mensal
SELECT nome, email
FROM cliente
-- ativo=0 optou por não receber comunicações
WHERE ativo = 1;
Comentando CREATE TABLE
-- tabela de clientes cadastrados na locadora
CREATE TABLE cliente (
  cpf CHAR(14) PRIMARY KEY, -- formato '000.000.000-00'
  nome VARCHAR(100) NOT NULL,
  email VARCHAR(150) UNIQUE, -- usado para login
  cidade_cliente VARCHAR(80),
  ativo TINYINT DEFAULT 1 -- 1=ativo, 0=bloqueado para marketing
);
Comentando SELECT complexo com JOINs
-- relatório mensal: clientes que locaram Ação em outubro/2026
-- usado pela equipe de marketing para campanha de fidelização
SELECT c.nome, c.email,
       COUNT(*) AS locacoes_acao
FROM cliente c
INNER JOIN locacao l ON c.cpf = l.cpf_cliente
INNER JOIN filme f ON l.id_filme = f.id_filme
WHERE f.genero = 'Ação'
  AND MONTH(l.data_locacao) = 10 -- apenas outubro
  AND YEAR(l.data_locacao) = 2026
-- excluir clientes bloqueados para marketing
  AND c.ativo = 1
GROUP BY c.nome, c.email
ORDER BY locacoes_acao DESC;
Usando comentário de bloco para desativar código
-- versão ativa: filtro por cidade
SELECT * FROM cliente WHERE cidade_cliente = 'Umuarama';

/*
-- versão desativada: estava filtrando por ativo
-- removido em 28/10/2026 — novo requisito inclui inativos
SELECT * FROM cliente WHERE ativo = 1;
*/
💡 Comentário desatualizado é pior que nenhum: um comentário que dizia "só clientes ativos" mas a query foi alterada para incluir inativos sem atualizar o comentário vai enganar quem ler o código. Sempre que alterar a query, revise os comentários ao redor.

Comentar DDL
Script sem documentação — entregue ao grupo
CREATE TABLE locacao (

  cpf_cliente CHAR(14) NOT NULL,
  id_filme INT NOT NULL,

Gabarito esperado
-- registros de cada locação realizada na loja
CREATE TABLE locacao (
  id_locacao INT AUTO_INCREMENT PRIMARY KEY, -- gerado automaticamente
  cpf_cliente CHAR(14) NOT NULL, -- quem alugou
  id_filme INT NOT NULL, -- o que foi alugado
  data_locacao DATE NOT NULL, -- data da retirada, não da devolução
  -- integridade: não permite locação de cliente ou filme inexistente
  FOREIGN KEY (cpf_cliente) REFERENCES cliente(cpf),
  FOREIGN KEY (id_filme) REFERENCES filme(id_filme)
);

Comentar SELECT + JOIN
Script sem documentação — entregue ao grupo
SELECT c.nome, c.email, COUNT(*) AS total
FROM cliente c
INNER JOIN locacao l ON c.cpf = l.cpf_cliente
WHERE c.ativo = 1
  AND YEAR(l.data_locacao) = YEAR(CURDATE())
GROUP BY c.nome, c.email
HAVING COUNT(*) >= 5
ORDER BY total DESC;
Gabarito esperado
-- clientes elegíveis para o programa de fidelidade do ano corrente
-- critério: 5 ou mais locações no ano · apenas clientes ativos
SELECT c.nome, c.email, COUNT(*) AS total
FROM cliente c
INNER JOIN locacao l ON c.cpf = l.cpf_cliente
WHERE c.ativo = 1 -- bloqueados não participam
  AND YEAR(l.data_locacao) = YEAR(CURDATE()) -- apenas o ano atual
GROUP BY c.nome, c.email
HAVING COUNT(*) >= 5 -- mínimo definido pelo programa
ORDER BY total DESC;
4
Identifique e corrija os comentários ruins no script abaixo — remova os que repetem o código e substitua pelos que agregam valor real
Revisar comentários ruins
Script com comentários ruins — entregue ao grupo
-- faz um select na tabela filme
SELECT titulo, genero, ano
-- da tabela filme
FROM filme
-- filtra pelo gênero
WHERE genero = 'Ação'
-- ordena por ano
ORDER BY ano DESC;
-- fim do select
Gabarito esperado — versão corrigida
-- catálogo de filmes de Ação ordenado do mais recente ao mais antigo
-- usado na tela de destaques do site da locadora
SELECT titulo, genero, ano
FROM filme
WHERE genero = 'Ação'
ORDER BY ano DESC;
✅ Ponto pedagógico — rodada 4: reforçar que remover comentários ruins deixa o código mais limpo e profissional. Um código bem nomeado raramente precisa de comentários linha a linha — o comentário entra apenas onde o código não consegue se explicar sozinho. O resultado da rodada 4 tem menos linhas de comentário e mais valor informativo.
