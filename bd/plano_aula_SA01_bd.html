📚 Conteúdo da Aula Fundamentos

Hierarquia: Dado → Informação → Banco de Dados

🔢
Dado
"Fato bruto sem contexto — ainda não tem significado por si só"
  • 23, "João", true, 2026-02-11
  • Número, texto, booleano, data
  • Existe independente de interpretação
  • Ex: a temperatura 37 isolada não diz nada
💡
Informação
"Dado processado com significado e contexto — útil para quem recebe"
  • João tem 23 anos
  • A febre do paciente é 37,8 °C
  • Depende de quem recebe e do contexto
  • Dado + significado + contexto = informação
🗄️
Banco de Dados
"Coleção organizada de dados inter-relacionados, armazenados de forma persistente"
  • Guarda dados de forma estruturada
  • Permite consultar, inserir, alterar, excluir
  • Gerenciado por um SGBD
  • Sustenta toda aplicação digital

Características de um Banco de Dados

💾
Persistência
Dados permanecem armazenados mesmo após o sistema ser encerrado
👥
Compartilhamento
Múltiplos usuários e sistemas acessam os mesmos dados simultaneamente
✅
Integridade
Dados corretos, consistentes e confiáveis em todo momento
🔒
Segurança
Controle de quem pode acessar e modificar os dados
🔌
Independência
Dados separados dos programas que os utilizam

Banco de Dados × Planilha Eletrônica

🗄️
Banco de Dados
  • Múltiplos usuários simultâneos sem conflito
  • Controle de acesso por perfil (admin, leitura…)
  • Garante integridade com regras automáticas
  • Suporta volumes enormes de dados
  • Permite consultas complexas com SQL
Use quando: há vários usuários, regras de negócio e volume crescente de dados
📊
Planilha Eletrônica
  • Ideal para análise e cálculos pontuais
  • Fácil de usar — sem instalação de SGBD
  • Ótima para visualização e gráficos
  • Limitada para múltiplos usuários simultâneos
  • Sem controle de integridade automático
Use quando: uso pessoal, poucos dados e análises simples sem concorrência

🎮 Dinâmica Onde Está o Banco de Dados?
🗄️ Onde Está o Banco de Dados?
⏱️ Duração: 15 minutos
👥 Agrupamento: grupos de 4 a 5 alunos
🎯 Objetivo: identificar BDs em situações reais e conectar ao papel do desenvolvedor
1
Distribuição das situações (1 min) — Cada grupo recebe 4 cartões (ou vê os slides projetados) com situações do cotidiano digital
2
Análise em grupo (7 min) — O grupo discute cada situação e responde: "Tem banco de dados aqui? O que ele guarda?"
3
Compartilhamento (7 min) — Cada grupo apresenta uma situação; professor complementa e reforça os conceitos de persistência e compartilhamento
Situação do cotidianoTem BD?O que guarda
Fazer login no Instagram SIM Usuário, senha, seguidores, publicações, mensagens
Pagar com cartão no supermercado SIM Transações, saldo, histórico, dados do cliente
Abrir um arquivo de texto no computador NÃO Arquivo local — não há sistema de BD
Pedir um lanche no app de delivery SIM Pedido, endereço, cardápio, pagamento, histórico
Usar uma calculadora física NÃO Sem armazenamento — apenas processamento local
Acessar a nota no portal do aluno SIM Notas, frequência, dados do aluno, disciplinas
Assistir a um vídeo no YouTube SIM Histórico de visualizações, preferências, dados do canal
Editar uma planilha offline no Excel NÃO Arquivo local — sem BD relacional por baixo
💡 Reforço pedagógico: o objetivo não é decorar os exemplos, mas desenvolver o raciocínio — "se um sistema precisa guardar dados para usar depois, ele tem um banco de dados por baixo." Reforçar que o papel do desenvolvedor inclui projetar e manter esses bancos.
