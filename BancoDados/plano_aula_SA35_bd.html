🏆 Objetivos de Aprendizagem 6 objetivos

Ao final do projeto, o grupo será capaz de:

  • 1Identificar as entidades, atributos e relacionamentos de um cenário real e representá-los em um DER
  • 2Converter o DER em modelo físico, criando scripts DDL com todas as tabelas, PKs e FKs
  • 3Popular o banco de dados com registros coerentes usando INSERT, respeitando a ordem de dependência das FKs
  • 4Executar consultas com INNER JOIN, LEFT JOIN, GROUP BY e HAVING para responder perguntas de negócio
  • 5Criar pelo menos uma VIEW e uma operação de UPDATE ou DELETE envolvida em transação (BEGIN / COMMIT)
  • 6Documentar todos os scripts com cabeçalho de bloco e comentários nas decisões não-óbvias
🏥 Cenário do Projeto Clínica Veterinária "Pet Saúde"
Briefing do Projeto
Clínica Veterinária "Pet Saúde"
A clínica veterinária Pet Saúde atende cães e gatos de donos cadastrados. Cada animal tem um dono e pode passar por múltiplas consultas ao longo do tempo. Em cada consulta, um veterinário registra o diagnóstico, o valor cobrado e os procedimentos realizados. O sistema precisa responder perguntas como: quais animais têm mais consultas?, quais procedimentos foram mais realizados?, quais donos ainda não trouxeram seus animais em 2026?

O grupo deverá modelar, implementar e consultar o banco de dados deste sistema do zero, integrando todos os conteúdos do 3º Trimestre.

Entidades e atributos sugeridos
👤 dono
cpf CHAR(14) PK
nome VARCHAR(100)
telefone VARCHAR(20)
email VARCHAR(150)
cidade VARCHAR(80)
ativo TINYINT DEFAULT 1
1 → N com animal
🐾 animal
id_animal INT PK AUTO
nome_animal VARCHAR(60)
especie VARCHAR(40)
raca VARCHAR(60)
cpf_dono CHAR(14) FK
dt_nascimento DATE
N → 1 com dono · 1 → N com consulta
👨‍⚕️ veterinario
crmv CHAR(10) PK
nome VARCHAR(100)
especialidade VARCHAR(60)
1 → N com consulta
📋 consulta
id_consulta INT PK AUTO
id_animal INT FK
crmv CHAR(10) FK
data_consulta DATE
diagnostico TEXT
valor DECIMAL(8,2)
N → 1 com animal e veterinario
💉 procedimento
id_proc INT PK AUTO
nome_proc VARCHAR(100)
descricao TEXT
custo DECIMAL(8,2)
N → N com consulta via associativa
🔗 consulta_proc
id_consulta INT FK
id_proc INT FK
PRIMARY KEY (id_consulta, id_proc)
Tabela associativa N:N
Relacionamentos do modelo
  • 1 : N Um dono pode ter vários animais
  • 1 : N Um animal pode ter várias consultas
  • 1 : N Um veterinário pode realizar várias consultas
  • N : N Uma consulta pode ter vários procedimentos e um procedimento pode aparecer em várias consultas — resolvido pela tabela consulta_proc
⚠️ Liberdade de modelagem: o grupo pode adicionar ou remover atributos, renomear colunas e incluir novas entidades desde que o modelo faça sentido para o negócio descrito e seja devidamente documentado. O professor deve ser consultado antes de mudanças estruturais significativas.
🔨 Fases do Projeto 4 fases + documentação
01 Modelagem e DDL ~20 min
Converter o cenário em modelo físico com scripts SQL.
  • Esboçar o DER (papel ou digital)
  • Definir PKs, FKs e tipos de dado
  • Escrever CREATE TABLE para todas as 6 tabelas
  • Respeitar a ordem de criação (sem FK órfã)
  • Adicionar cabeçalho de bloco ao script
02 Carga de Dados — INSERT ~15 min
Popular o banco com dados suficientes para as consultas.
  • Ao menos 5 donos e 8 animais
  • 3 veterinários e 5 procedimentos
  • 10+ consultas com datas variadas
  • Registros em consulta_proc (N:N)
  • Inserir na ordem: dono → animal → vet → consulta
03 Consultas — SELECT e VIEW ~25 min
Responder às perguntas de negócio com SQL.
  • INNER JOIN: consultas com nome do animal e veterinário
  • LEFT JOIN: animais sem nenhuma consulta
  • GROUP BY + HAVING: top animais por nº de consultas
  • Subquery: donos acima da média de animais
  • CREATE VIEW: vw_historico_consultas
04 DML Avançado + Documentação ~15 min
Operações seguras e documentação final.
  • UPDATE com transação: registrar novo diagnóstico
  • DELETE com transação: remover animal sem consultas
  • Comentários em todas as queries complexas
  • Revisar e corrigir comentários óbvios
  • Organizar os scripts para a apresentação
Exemplo DDL — script comentado (referência)
/*
  Banco: pet_saude
  Script: DDL — criação das tabelas
  Grupo: [nomes] · Data: 04/11/2026
*/

-- donos cadastrados na clínica (ativo=0 = descadastrado)
CREATE TABLE dono (
  cpf CHAR(14) PRIMARY KEY, -- formato '000.000.000-00'
  nome VARCHAR(100) NOT NULL,
  telefone VARCHAR(20),
  email VARCHAR(150) UNIQUE,
  cidade VARCHAR(80),
  ativo TINYINT DEFAULT 1
);

-- animais atendidos (vínculo obrigatório com um dono cadastrado)
CREATE TABLE animal (
  id_animal INT AUTO_INCREMENT PRIMARY KEY,
  nome_animal VARCHAR(60) NOT NULL,
  especie VARCHAR(40),
  raca VARCHAR(60),
  cpf_dono CHAR(14) NOT NULL,
  dt_nascimento DATE,
  FOREIGN KEY (cpf_dono) REFERENCES dono(cpf)
);

💡 Consultas Obrigatórias — Desafios DQL 5 consultas · todas obrigatórias
  • 1
    Liste todas as consultas com: nome do animal, espécie, nome do dono e nome do veterinário — ordenadas pela data mais recente
    INNER JOIN
    -- histórico de consultas com dados completos de animal, dono e veterinário
    SELECT a.nome_animal, a.especie,
           d.nome AS nome_dono,
           v.nome AS nome_vet,
           c.data_consulta, c.valor
    FROM consulta c
    INNER JOIN animal a ON c.id_animal = a.id_animal
    INNER JOIN dono d ON a.cpf_dono = d.cpf
    INNER JOIN veterinario v ON c.crmv = v.crmv
    ORDER BY c.data_consulta DESC;
  • 2
    Liste todos os animais que nunca tiveram nenhuma consulta — com o nome e cidade do seu dono
    LEFT JOIN + IS NULL
    -- animais nunca atendidos — candidatos a campanha de retorno
    SELECT a.nome_animal, a.especie,
           d.nome AS nome_dono, d.cidade
    FROM animal a
    INNER JOIN dono d ON a.cpf_dono = d.cpf
    LEFT JOIN consulta c ON a.id_animal = c.id_animal
    WHERE c.id_consulta IS NULL;
  • 3
    Liste os animais com mais de 2 consultas, exibindo nome, espécie e o total de consultas — ordenados do maior para o menor
    GROUP BY + HAVING
    -- animais frequentes — possíveis candidatos ao plano de saúde
    SELECT a.nome_animal, a.especie,
           COUNT(*) AS total_consultas
    FROM animal a
    INNER JOIN consulta c ON a.id_animal = c.id_animal
    GROUP BY a.id_animal, a.nome_animal, a.especie
    HAVING COUNT(*) > 2
    ORDER BY total_consultas DESC;
  • 4
    Liste os donos que possuem mais animais que a média de animais por dono — exibindo nome e total de animais
    Subquery
    -- donos com tutela acima da média — prospecção para plano família
    SELECT d.nome, COUNT(*) AS qtd_animais
    FROM dono d
    INNER JOIN animal a ON d.cpf = a.cpf_dono
    GROUP BY d.cpf, d.nome
    HAVING COUNT(*) > (
                  SELECT AVG(qtd)
                  FROM (
                   SELECT COUNT(*) AS qtd
                   FROM animal
                   GROUP BY cpf_dono
                  ) sub
                )
    ORDER BY qtd_animais DESC;
  • 5
    Crie uma VIEW chamada vw_historico_consultas com nome do animal, nome do dono, veterinário, data e valor — depois consulte a view filtrando por um veterinário específico
    CREATE VIEW
    -- view reutilizável para relatórios e sistemas externos
    CREATE OR REPLACE VIEW vw_historico_consultas AS
    SELECT a.nome_animal, a.especie,
           d.nome AS nome_dono,
           v.nome AS nome_vet, v.especialidade,
           c.data_consulta, c.diagnostico, c.valor
    FROM consulta c
    INNER JOIN animal a ON c.id_animal = a.id_animal
    INNER JOIN dono d ON a.cpf_dono = d.cpf
    INNER JOIN veterinario v ON c.crmv = v.crmv;

    -- uso da view com filtro por veterinário
    SELECT * FROM vw_historico_consultas
    WHERE nome_vet = '[nome do vet do grupo]';
🚀 Desafio extra (para grupos que concluírem cedo): usar BEGIN / COMMIT para registrar a devolução de uma consulta — UPDATE no diagnóstico + INSERT em consulta_proc na mesma transação, com ROLLBACK caso um dos procedimentos não exista.
