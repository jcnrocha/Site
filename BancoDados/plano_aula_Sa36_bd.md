# UC: Fundamentos de Banco de Dados
## SA 36 — 3º Trimestre

---

## Tema

Apresentação do Projeto Integrador + Revisão Geral

---

## Objetivos de Aprendizagem

Ao final desta aula, o aluno será capaz de:

- Apresentar oralmente o Projeto Integrador com clareza, domínio técnico e participação de todos os integrantes
- Demonstrar a integração dos conteúdos do ano: DER, modelo lógico, modelo físico, SQL e documentação
- Avaliar as apresentações dos colegas com base nos critérios definidos no enunciado do projeto
- Revisar os principais conteúdos do 3º trimestre de forma conectada e progressiva — preparando-se para a prova
- Identificar os tópicos que exigem maior atenção individual antes da avaliação

**Habilidades da matriz:** H03 · H04 · H06 · H07 · H08

---

## Conteúdo

**Parte 1 — Apresentação do Projeto Integrador:**
- Cada grupo apresenta o sistema desenvolvido: cenário, decisões de modelagem, script SQL e consultas
- A apresentação deve cobrir: DER, modelo lógico, modelo físico, pelo menos 2 consultas SQL demonstradas

**Parte 2 — Revisão Geral do 3º Trimestre:**
- SQL DQL — SELECT: estrutura, WHERE, ORDER BY, DISTINCT e operadores
- Funções de Agregação: COUNT, SUM, AVG, MIN, MAX · GROUP BY · HAVING · diferença do WHERE
- JOINs: INNER JOIN, LEFT JOIN, RIGHT JOIN · IS NULL para encontrar ausências
- Subqueries: no WHERE com comparação, IN e NOT IN · subquery no FROM
- Views: CREATE VIEW, DROP VIEW · vantagens e limitações
- SQL DML Avançado: UPDATE e DELETE com WHERE · transações BEGIN/COMMIT/ROLLBACK
- Documentação: comentários de linha `--` e de bloco `/* */` · boas práticas

## Dinâmica — Apresentação do Projeto Integrador

**Duração:** conforme o número de grupos · **Agrupamento:** grupos formados na SA 35

**Objetivo:** consolidar o aprendizado do ano por meio da apresentação pública de um banco de dados real — desenvolvendo comunicação oral, domínio técnico e capacidade de justificar decisões de modelagem e SQL.

**Estrutura de cada apresentação:**

| Parte | Conteúdo esperado | Tempo |
|-------|-------------------|-------|
| 1 | Apresentação do cenário e das decisões de modelagem — por que fizeram cada escolha | 3 min |
| 2 | DER e modelo lógico — exibir e explicar as entidades, relacionamentos e chaves | 3 min |
| 3 | Script SQL — mostrar o CREATE TABLE de pelo menos 2 tabelas com comentários | 2 min |
| 4 | Consultas SQL — demonstrar ao menos 2 queries com JOIN ou GROUP BY | 3 min |
| 5 | Perguntas da turma e do professor | 2 min |

**Critérios de avaliação (Peso 20):**

| Critério | Pontos |
|----------|--------|
| Modelo Conceitual — DER correto e completo | 4 |
| Modelo Lógico e Físico — script SQL funcional e bem estruturado | 4 |
| Population e consultas SQL — dados coerentes e consultas funcionais | 4 |
| Documentação — comentários no código SQL | 4 |
| Apresentação oral — clareza, domínio e participação de todos | 4 |
| **Total** | **20** |

---

## Avaliação

Esta aula não possui questionário individual. A avaliação formal é o Projeto Integrador com peso 20, conforme critérios acima. A participação oral durante a revisão geral pode ser registrada como atividade de sala conforme critério do professor.
