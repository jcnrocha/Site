# UC: Fundamentos de Banco de Dados
## SA 01 — 1º Trimestre

---

## Tema

Introdução ao Banco de Dados

---

## Objetivos de Aprendizagem

Ao final da aula, o aluno será capaz de:

- Compreender o que é um banco de dados e qual problema ele resolve no contexto de sistemas de informação
- Diferenciar dado, informação e banco de dados — entendendo a hierarquia entre os conceitos
- Identificar as principais características de um banco de dados: persistência, compartilhamento, integridade e segurança
- Reconhecer situações do cotidiano em que bancos de dados estão presentes, inclusive em sistemas de tecnologia
- Diferenciar banco de dados de planilha eletrônica — vantagens e limitações de cada abordagem

**Habilidade da matriz:** H01 — Identificar conceito, tipos, características e armazenamento do banco de dados

---

## Conteúdo

- Apresentação da UC: o que é Fundamentos de Banco de Dados, o que será visto no ano e como a disciplina se conecta com as demais UCs do curso
- O que é um dado — definição e exemplos
- O que é uma informação — dado processado com significado e contexto
- O que é um banco de dados — definição formal e informal
- Por que os bancos de dados existem: o problema de guardar e recuperar dados de forma confiável
- Características de um banco de dados:
  - **Persistência** — os dados permanecem armazenados mesmo após o encerramento do sistema
  - **Compartilhamento** — múltiplos usuários e sistemas podem acessar os mesmos dados simultaneamente
  - **Integridade** — os dados devem ser corretos, consistentes e confiáveis
  - **Segurança** — controle de quem pode acessar e modificar os dados
  - **Independência** — os dados são separados dos programas que os utilizam
- Banco de dados × planilha eletrônica: quando cada um é adequado
- Exemplos de bancos de dados no cotidiano: sistemas bancários, e-commerce, redes sociais, sistemas de saúde, aplicativos de entrega
- Exemplos de bancos de dados no contexto de desenvolvimento de sistemas: cadastro de usuários, controle de estoque, sistema de pedidos

## Dinâmica — "Onde Está o Banco de Dados?"

**Duração:** 15 minutos · **Agrupamento:** grupos de 4 a 5 alunos

**Objetivo:** identificar a presença de bancos de dados em situações reais do cotidiano dos alunos, desenvolvendo a percepção de que toda aplicação digital depende de um mecanismo de armazenamento de dados — e conectando esse conceito ao papel do desenvolvedor de sistemas.

**Materiais:** cartões com situações do cotidiano — um conjunto por grupo · ou slide com as situações projetadas.

**Passo a passo:**

1. **Distribuição das situações** *(1 min)*
Cada grupo recebe 4 cartões com situações do cotidiano digital. O desafio é responder para cada uma: *"Tem banco de dados aqui? O que ele guarda?"*

2. **Análise em grupo** *(7 min)*
O grupo discute cada cartão e preenche: SIM ou NÃO tem banco de dados, e quais dados são armazenados naquela situação.

3. **Compartilhamento** *(7 min)*
Cada grupo apresenta uma situação. O professor complementa, corrige e reforça os conceitos de persistência e compartilhamento em cada caso.

**Situações sugeridas:**

| Situação | Tem BD? | O que guarda |
|----------|---------|--------------|
| Fazer login no Instagram | Sim | Usuário, senha, seguidores, publicações, mensagens |
| Pagar com cartão no supermercado | Sim | Transações, saldo, histórico, dados do cliente |
| Abrir um arquivo de texto no computador | Não | Arquivo local — não há sistema de BD |
| Pedir um lanche no app de delivery | Sim | Pedido, endereço, cardápio, pagamento, histórico |
| Usar uma calculadora física | Não | Sem armazenamento — apenas processamento local |
| Acessar a nota no portal do aluno | Sim | Notas, frequência, dados do aluno, disciplinas |
| Assistir a um vídeo no YouTube | Sim | Histórico de visualizações, preferências, dados do canal |
| Editar uma planilha offline no Excel | Não | Arquivo local — sem BD relacional |

> **Nota:** o objetivo não é decorar os exemplos, mas desenvolver o raciocínio: *"se um sistema precisa guardar dados para usar depois, ele tem um banco de dados por baixo."* Reforçar que o papel do desenvolvedor de sistemas inclui projetar e manter esses bancos.
