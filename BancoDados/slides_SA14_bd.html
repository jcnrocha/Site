Antes de tudo: por que revisar?

🎯 SQL é a Ferramenta, o Banco é a Obra

SQL (Structured Query Language) é a linguagem que você usa para conversar com o banco de dados — criar, alterar, ler e proteger as informações. Não existe banco de dados relacional sem SQL por trás.

No projeto SQL constrói a Biblioteca
Cada tabela do sistema (LIVRO, AUTOR, MEMBRO, EMPRÉSTIMO...) nasce de um comando CREATE TABLE que você mesmo vai escrever.
Na vida real Toda empresa usa isso
Sistemas de biblioteca, e-commerce, bancos e escolas — todos guardam dados em tabelas criadas e populadas com SQL.

Por isso a revisão de hoje é séria: cada detalhe de sintaxe que você fixar agora evita retrabalho nas próximas 8 aulas do projeto.

Revisão · linguagem SQL

🎯 As Quatro Categorias do SQL

O SQL se divide em categorias segundo o que cada comando faz: define estrutura, manipula dados, consulta ou controla acesso. Hoje vamos a fundo em DDL e DML.

DDL Definição
CREATE, ALTER, DROP — molda a estrutura das tabelas.
DML Manipulação
INSERT, UPDATE, DELETE — altera os dados dentro das tabelas.
DQL Consulta
SELECT — busca e filtra os dados já armazenados.
DCL Controle
GRANT, REVOKE — define quem pode acessar o quê.

Hoje na prática: você vai usar DDL para criar as tabelas e DML para popular com dados — DQL e DCL ficam para mais adiante.

DDL · anatomia do comando

🔍 Anatomia de um CREATE TABLE

Todo CREATE TABLE segue a mesma estrutura: nome da tabela, e dentro dos parênteses, uma lista de colunas — cada uma com nome, tipo de dado e, opcionalmente, restrições.

CREATE TABLE membro (
  id_membro  INT        PRIMARY KEY,
  nome       VARCHAR(100) NOT NULL,
  matricula   CHAR(10)   UNIQUE
);
Nome da coluna + tipo
id_membro INT — define o nome e que tipo de valor aquela coluna aceita.
Restrição (opcional)
PRIMARY KEY, NOT NULL, UNIQUE — regras extras de integridade para aquela coluna.

CREATE TABLE · restrições de integridade

🛡️ Restrições — As Regras do Banco

Restrições (constraints) impedem que dados inválidos entrem na tabela. Sem elas, o banco aceita qualquer coisa — inclusive erros.

PRIMARY KEY
Identifica cada linha de forma única. Não pode repetir, não pode ser nula.
FOREIGN KEY
Liga uma tabela a outra, referenciando a chave primária dela.
NOT NULL
Obriga a coluna a ter sempre um valor — não aceita vazio.
UNIQUE / DEFAULT
UNIQUE impede repetição; DEFAULT define um valor automático quando nada é informado.

Dica: toda tabela do sistema da Biblioteca vai precisar de pelo menos uma PRIMARY KEY — sem ela, não há como identificar um registro específico.

Exemplo guiado · construindo a tabela MEMBRO

📋 Passo a Passo: Criando a Tabela MEMBRO

Vamos construir juntos a tabela de membros (leitores) da Biblioteca, decidindo cada coluna com intenção.

01
id_membro INT PRIMARY KEY — todo membro precisa de um identificador único.
02
nome VARCHAR(100) NOT NULL — todo membro tem nome, e ele nunca pode ficar em branco.
03
matricula CHAR(10) UNIQUE — a matrícula identifica o aluno na escola e não pode repetir.
04
email VARCHAR(120) — útil para contato, mas não obrigatório nem único aqui.
05
telefone VARCHAR(15) — opcional, sem restrição extra.
Exemplo guiado · resultado final

💻 MEMBRO Pronta — e a Chegada da FOREIGN KEY

Eis a tabela completa que construímos passo a passo. Agora veja como uma segunda tabela pode referenciar a primeira com FOREIGN KEY.

✔ Tabela MEMBRO
CREATE TABLE membro (
  id_membro INT PRIMARY KEY,
  nome VARCHAR(100) NOT NULL,
  matricula CHAR(10) UNIQUE,
  email VARCHAR(120),
  telefone VARCHAR(15)
);
✔ Tabela EMPRESTIMO referenciando MEMBRO
CREATE TABLE emprestimo (
  id_emprestimo INT PRIMARY KEY,
  id_membro INT REFERENCES membro(id_membro),
  data_saida DATE NOT NULL
);

Conclusão: a FOREIGN KEY garante que um empréstimo só pode referenciar um membro que realmente existe na tabela MEMBRO.

DML · povoando a tabela

🎯 INSERT INTO — Colocando Dados na Tabela

Depois de criar a estrutura com CREATE TABLE, usamos INSERT INTO para adicionar registros reais — cada execução cria uma ou mais linhas na tabela.

INSERT INTO membro (id_membro, nome, matricula)
VALUES (1, 'Ana Lima', '2026A001');
Tabela + colunas
INSERT INTO membro (...) — diz em qual tabela e em quais colunas os dados vão entrar.
Valores correspondentes
VALUES (...) — cada valor precisa estar na mesma ordem e ser do mesmo tipo da coluna.
INSERT INTO · duas formas de inserir

💻 INSERT INTO — Com e Sem Colunas

Sem a lista de colunas, a ordem dos valores precisa seguir exatamente a ordem em que as colunas foram criadas na tabela — um erro fácil de cometer.

✗ Sem lista de colunas
INSERT INTO membro
VALUES (2, 'João Silva', '2026A002',
    'joao@email.com', '4199990000');
-- precisa bater 100% com a ordem da tabela
✔ Com lista de colunas
INSERT INTO membro
(id_membro, nome, matricula)
VALUES (2, 'João Silva', '2026A002');
-- email e telefone ficam NULL automaticamente

Conclusão: listar as colunas deixa o comando mais legível, mais seguro e permite omitir colunas opcionais.

INSERT INTO · múltiplos registros

📋 Inserindo Vários Registros de Uma Vez

Um único INSERT INTO pode popular várias linhas — basta separar os grupos de valores por vírgula. Mais rápido e mais organizado que repetir o comando.

01
Escolha a tabela e as colunas — defina exatamente quais colunas vão receber valores.
02
Abra o primeiro grupo de valores — um (...) representa uma linha completa.
03
Separe os grupos por vírgula — cada novo parêntese é um novo registro na mesma instrução.
04
Confira ordem e tipos — cada valor deve corresponder exatamente ao tipo da coluna.
05
Finalize com ponto e vírgula — só então execute o comando completo.
Exemplo completo · vários membros de uma vez

💻 INSERT em Lote — 3 Membros da Biblioteca

Veja como inserir três membros em um único comando — é exatamente o que você vai entregar na atividade de hoje.

INSERT INTO membro (id_membro, nome, matricula)
VALUES
  (1, 'Ana Lima', '2026A001'),
  (2, 'João Silva', '2026A002'),
  (3, 'Maria Souza', '2026A003');

Atenção: só o último grupo de valores termina com ponto e vírgula — os anteriores terminam com vírgula.

Erros comuns no INSERT INTO

⚖️ Errado × Certo

✗ Erros frequentes
✗ Valor de texto em campo INT
✗ Coluna NOT NULL sem valor
✗ Quantidade de valores diferente da de colunas
✗ Ordem dos valores trocada (sem lista de colunas)
✔ Como evitar
✔ Confira o tipo de cada coluna antes de inserir
✔ Sempre preencha campos obrigatórios
✔ Conte colunas e valores antes de executar
✔ Liste as colunas explicitamente no comando

Conclusão: a maioria dos erros de INSERT desaparece só conferindo tipo, ordem e obrigatoriedade das colunas antes de rodar o comando.

Análise de erro · do problema à correção

💻 Encontrando e Corrigindo um Erro Real

A tabela livro tem ano_publicacao INT NOT NULL. Veja o que acontece quando o INSERT ignora essas regras.

✗ Comando com erro
INSERT INTO livro
(id_livro, titulo, ano_publicacao)
VALUES (1, 'Dom Casmurro', 'mil oitocentos');
-- texto em coluna INT
✔ Comando corrigido
INSERT INTO livro
(id_livro, titulo, ano_publicacao)
VALUES (1, 'Dom Casmurro', 1899);

Conclusão: o banco rejeita o valor incompatível com o tipo da coluna — isso é a integridade do banco te protegendo de um dado inválido.
